# Backlog 维护规则

## 状态值

仅使用以下状态值：

```
未开始    — 尚未开始实现
进行中    — 当前正在实现
已完成    — 实现与本地验证已完成
```

## 优先级

```
P0 — 阻塞当前阶段主链路
P1 — 重要但不阻塞主链路
P2 — 体验、质量或工程优化
P3 — 远期增强
```

## 验收状态

独立于开发状态：
```
未验收  — 尚未验收
待验收  — 实现完成，等待验收
已验收  — 通过阶段验收
验收未通过 — 验收发现缺陷
```

## Backlog Item 格式

```markdown
## BL-XXX-NNN 标题

- 模块：{模块名}
- 优先级：P0/P1/P2/P3
- 状态：未开始/进行中/已完成
- 当前阶段：阶段一
- 价值：{一句话描述业务价值}
- 关联模块文档：docs/modules/NN-xxx模块.md
- 关联计划：
- 关联代码：
- 验收状态：未验收

### 范围
- ...

### 不做
- ...

### 验收条件
- [ ] 具体可测的条件
- [ ] 不是 "功能正常"

### 依赖
- ...

### 状态记录
- YYYY-MM-DD：...
```

## 维护规则

1. 实现前：状态改为 `进行中`，创建 `plans/` 计划文件。
2. 实现后：状态改为 `已完成`，记录验证证据，同步 CHANGELOG。
3. 新需求先走 Mode B0（需求澄清）或 Mode B（backlog 生成）。
4. 高风险项（P0 + 跨模块/DB/安全/>20 文件）先走 Mode H（共识规划）。
5. 不新增 backlog 状态值，阻塞/搁置原因记录在状态记录中。
