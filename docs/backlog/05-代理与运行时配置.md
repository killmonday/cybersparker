## BL-RUNTIME-002 ceye DNSLog 配置与运行时集成

- 模块：代理与运行时配置
- 优先级：P0
- 状态：已完成
- 当前阶段：阶段三增强
- 价值：安全测试人员可以验证需要外带数据（OOB）的漏洞（如 SSRF、Blind RCE、SQL 外带注入），填补当前 Nuclei 模板执行的最大能力缺口。
- 关联模块文档：docs/modules/05-代理与运行时配置模块.md
- 关联计划：
- 关联代码：`app_cybersparker/models.py`（新增 CeyeConfig）, `app_cybersparker/views/expload/`（新增 ceye_config 视图）, `app_cybersparker/views/expload/task_manage/nuclei_runtime_engine.py`（注入 ceye client）
- 验收状态：已验收

### 范围

- 新增 `CeyeConfig` 模型（`api_token` + `identifier`），全局单条记录

## BL-PROXY-002 任务系统代理统一控制

- 模块：代理与运行时配置
- 优先级：P0
- 状态：已完成
- 当前阶段：当前
- 价值：用户在任务配置中选择是否使用代理及使用哪个代理后，该配置在所有任务阶段（Web 扫描、漏洞扫描）中统一生效。任务配置"不使用代理"时强制直连不走 OS 环境变量代理。同一进程内多个任务并发的代理互不干扰。
- 关联模块文档：docs/modules/05-代理与运行时配置模块.md
- 关联计划：plans/2026-06-06-任务系统代理统一控制.md
- 关联代码：`app_cybersparker/lib/request_runtime/patch/hook_request.py`, `app_cybersparker/views/expload/task_manage/batch_task_executor.py`, `app_cybersparker/views/expload/task_manage/auto_exp_task.py`, `app_cybersparker/services/dirscan_worker.py`, `app_cybersparker/views/expload/fingerPrint_debug.py`, `app_cybersparker/services/request_runtime_config_service.py`, `app_cybersparker/CLAUDE.md`, `app_cybersparker/tests.py`
- 验收状态：已验收

### 范围

- hook_request.py：用 `threading.local()` 替代 `conf.proxies`，sentinel 模式（`None`=未设置，`{}`=设置空）区分任务线程和非任务线程
- batch_task_executor.py：删除 save/restore 逻辑，`run()` + `consumer_exp()` 两个入口设 `set_task_proxy()`
- auto_exp_task.py：`run()` + `exp_consumer()` 两个入口设 `set_task_proxy()`
- dirscan_worker.py：Phase 2 `consumer()` 子线程入口设 `set_task_proxy()`，重构 proxy 传递
- fingerPrint_debug.py：`trust_env=False` 防止穿透到 OS 环境变量代理
- single_task_executor.py：删除死代码
- request_runtime_config_service.py：`conf.proxies` 赋值标记废弃
- tests.py：删除 2 旧测试、新增 6 新测试

### 不做

- 不删 `conf.proxies` 定义和 `refresh_conf_from_db()` 中的赋值
- 不改 `cyberspace_engine_adapters.py` 的独立代理体系
- 不改 aiohttp 路径的代理控制（auto_exp_task Phase 1 / dirscan Phase 1 已有各自控制）

### 验收条件

- [ ] hook：任务线程设 `set_task_proxy()` 后，插件内 `requests.get()` 走任务代理
- [ ] hook：任务线程设 `set_task_proxy({})` 时，插件传了显式代理也被忽略，走直连
- [ ] hook：非任务线程不设 `set_task_proxy()`，显式传 proxies 保留原行为
- [ ] hook：非任务线程不设 `set_task_proxy()`，`proxies=None` 回退到 `conf.proxies`（向后兼容）
- [ ] 批量任务、自动扫描任务、目录扫描任务：consumer 子线程入口设 `set_task_proxy` 后 POC 请求走任务代理
- [ ] 指纹调试页：用户不选代理时不走 OS 环境变量代理
- [ ] 两个批量任务同一 Django 进程并发时各自走各自配置的代理，互不覆盖
- [ ] socks5 代理发起 HTTP 请求，响应 200 且目标服务器确认来源 IP 为代理 IP（requests 和 aiohttp 两侧均验证）

### 依赖

- `PySocks` + `aiohttp-socks`（已在 `requirements.txt` 中声明）
- `request_runtime_config_service.py:_build_proxy_from_setting()` — 统一构造代理 dict 的工具函数
