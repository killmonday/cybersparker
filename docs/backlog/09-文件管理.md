# 09-文件管理

## BL-FILE-001 文件管理页（上传/下载/删除）

- 模块：文件管理
- 优先级：P1
- 状态：已完成
- 当前阶段：阶段十四
- 价值：用户可独立管理目标文件，无需通过任务表单间接操作；文件能被任务表单的"历史上传文件"直接选用。
- 关联模块文档：docs/modules/09-文件管理模块.md
- 关联计划：
- 关联代码：
- 验收状态：已验收

### 范围

- 新建 React 页面 `/react-shell/target-files`，侧边栏新增入口（`SidebarLayout.tsx` menuItems + PAGE_TITLES + `App.tsx` 路由）
- 后端 API（文件列表、上传、下载、单个删除、批量删除），所有 API 沿用现有 AuthMiddleware 会话认证
- 上传：前端 + 后端双重校验 `.txt` 后缀，上传大小限制 30MB
- 重名文件自动加数字后缀（`targets.txt` → `targets_1.txt`），并发安全（循环检查文件系统直到找到不冲突的后缀）
- 下载：`GET /api/v1/target-files/<filename>/download`，Content-Disposition attachment
- 删除：单个 + 批量。删除前检查 `auto_scan_tasks.history_files` 和 `batch_EXPTask.history_files` 是否包含该文件名。如被引用则弹出警告（列明任务 ID），用户确认后：删除文件 + 从所有引用任务的 `history_files` 字段中移除该文件名
- 文件列表：文件名、大小（B/KB/MB）、行数（非空行）、上传时间（mtime），支持按文件名/大小/时间排序（前端排序），默认时间倒序
- **路径安全**：所有接受文件名的 API 必须在 `get_absolute_target_path()` 之后增加第二重校验：解析路径必须在 `EXP_input/` 目录内，且不在任何子目录下（防御 `../` 路径穿越）

### 不做

- 不建数据库模型
- 不管理子目录（`.merged/`、`engine_assets/`）
- 不支持 .txt 以外的后缀
- 不校验文件内容（不检查每行是否为合法 URL/域名/IP）

### 验收条件

- [ ] 侧边栏"系统配置"分组下出现"文件管理"入口，点击进入 `/react-shell/target-files`
- [ ] 文件列表正确展示文件名、大小、行数、上传时间，支持按文件名/大小/时间点击排序，默认时间倒序
- [ ] 上传 .txt 文件（≤30MB）成功，文件出现在列表和任务表单"历史上传文件"中
- [ ] 上传非 .txt 文件时前端拒绝并提示"仅支持 .txt 后缀的文本文件"
- [ ] 上传同名文件自动加后缀重命名（并发安全），不覆盖已有文件
- [ ] 上传超过 30MB 的文件后端返回 400
- [ ] 点击下载按钮，浏览器触发下载
- [ ] 删除未被引用文件成功，文件从列表消失
- [ ] 删除被引用文件时弹出警告（列出引用任务 ID），用户确认后：文件删除 + 引用任务的 `history_files` 字段同步移除该文件名
- [ ] 批量勾选删除，逐文件汇总引用警告，用户确认后批量执行
- [ ] 构造 `../` 路径穿越请求（如 `GET /api/v1/target-files/../settings.py/download`），后端返回 400/404
- [ ] 未登录用户调用 API 返回 401
- [ ] 列表不展示 `engine_assets/` 和 `.merged/` 子目录下的文件
- [ ] 删除文件后，任务表单的"历史上传文件"下拉不再显示该文件

### 依赖

- 已有 `EXP_input/` 目录
- 已有 `get_absolute_target_path()` 路径安全校验（需增加第二重目录边界检查）
- 已有 AuthMiddleware 会话认证
- `HistoryFilePicker` 组件可复用样式参考，但新页面为独立功能
