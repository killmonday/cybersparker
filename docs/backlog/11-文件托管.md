## BL-HOST-001 文件托管页面（上传/下载/删除/重命名/公开鉴权）

- 模块：文件托管
- 优先级：P1
- 状态：已完成
- 当前阶段：阶段十八
- 价值：用户可以通过后台方便地上传任意文件，生成直接下载链接分发给公网，无需额外搭建文件服务。
- 关联模块文档：docs/modules/11-文件托管模块.md
- 关联计划：plans/2026-06-19-文件托管.md
- 关联代码：app_cybersparker/views/expload/hosted_file_manage.py、frontend/src/pages/HostedFileListPage.tsx
- 验收状态：已验收

### 范围

- 新增数据库模型 `HostedFile`，记录文件名、磁盘文件名、大小、公开/鉴权标记、上传时间
- 后端 API（`/api/v1/hosted-files`）：列表、上传、删除、重命名、修改公开/鉴权状态
- 公开下载入口 `/files/<filename>`：公开文件无需登录即可下载，鉴权文件未登录返回 403
- AuthMiddleware 放行 `/files/` 前缀，下载视图自行判断权限
- React 前端页面 `/react-shell/hosted-files`：文件列表、上传（含公开/鉴权选择）、删除、重命名、修改访问级别
- 侧边栏新增一级菜单"文件托管"
- 文件存储到项目根目录 `./upload_files/`，无文件类型限制，单文件上限 200MB

- [ ] 可为文件添加备注（最多500字），表格列展示备注内容

### 不做

- 不分文件夹/目录结构（扁平存储）
- 不支持批量上传
- 不生成临时链接/过期链接/访问计数
- 不限制下载频率
- 不修改现有"文件管理"页面

### 验收条件

- [ ] 用户可在"文件托管"页面上传任意类型文件（≤200MB），上传时选择"公开"或"需登录"
- [ ] 文件列表显示文件名、大小、访问级别（公开/需登录）、上传时间
- [ ] 可删除文件（同时删除磁盘文件 + 数据库记录）
- [ ] 可重命名文件（改磁盘实际文件名，修改后数据库记录同步更新）
- [ ] 可修改已有文件的公开/鉴权状态
- [ ] 公开文件：`GET /files/<filename>` 不登录即可下载
- [ ] 鉴权文件：未登录访问 `GET /files/<filename>` 返回 403；已登录正常下载
- [ ] 重命名后旧下载链接失效（因为磁盘文件名变了）
- [ ] 侧边栏独立一级菜单"文件托管"，点击进入新页面
- [ ] 上传、删除、重命名操作有前端确认/提示

### 依赖

- PostgreSQL（新模型需 migration）
- AuthMiddleware（需新增 `/files/` 前缀放行）
- React 前端框架（antd 6.x）
