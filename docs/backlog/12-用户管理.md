## BL-USER-002 用户管理 API

- 模块：用户管理
- 优先级：P0
- 状态：已完成
- 当前阶段：阶段十九
- 价值：管理员可以通过页面管理用户账户，角色边界在后端严格执行。
- 关联模块文档：docs/modules/12-用户管理模块.md
- 关联计划：plans/2026-06-19-用户管理设计.md
- 关联代码：app_cybersparker/views/user_manage.py（新建）
- 验收状态：已验收

### 范围

- GET `/api/v1/users`：用户列表（super_admin 看全部，admin 仅看 role=user 的记录，user 返回 403）
- POST `/api/v1/users`：创建用户（JSON body: username, password, role）——admin 只能传 role=user
- DELETE `/api/v1/users/<id>`：删除用户（删 auth.User + UserProfile + 清除目标 session）
- PUT `/api/v1/users/<id>/role`：改角色（仅 super_admin，改后清除目标 session）
- PUT `/api/v1/users/<id>/password`：改他人密码（super_admin 任何人，admin 仅 user）
- PUT `/api/v1/users/me/password`：改自己密码（所有角色）
- 所有接口内的角色边界校验逻辑

### 不做

- 不改动现有任何 views 文件
- 不加邮箱/手机号字段

### 验收条件

- [ ] super_admin 可创建 admin 和 user
- [ ] admin 只能创建 user，传 role=admin 返回 403
- [ ] super_admin 删除任何用户（除自己外）成功
- [ ] admin 删除 super_admin 或其他 admin 返回 403
- [ ] admin 删除 user 成功
- [ ] user 调用 `/api/v1/users` 下全部 6 个接口均返回 403
- [ ] super_admin 改角色成功，目标用户 session 被清除
- [ ] admin 改角色返回 403
- [ ] admin 改 user 密码成功，改 super_admin/admin 密码返回 403
- [ ] 用户改自己密码成功
- [ ] 删除用户后目标用户 session 被清除

### 依赖

- BL-USER-001（UserProfile 模型 + session["info"] 格式）

## BL-USER-003 后端权限执行

- 模块：用户管理
- 优先级：P0
- 状态：已完成
- 当前阶段：阶段十九
- 价值：普通用户无法执行任何写操作，无法查看插件/指纹内容。
- 关联模块文档：docs/modules/12-用户管理模块.md
- 关联计划：plans/2026-06-19-用户管理设计.md
- 关联代码：app_cybersparker/permissions.py（新建）、app_cybersparker/views/expload/ 下各 views 文件
- 验收状态：已验收

### 范围

- **Step 1 — 生成 API 盘点清单**：逐模块遍历 `views/expload/` 下所有 views 文件，列出全部 API 视图函数，标记每个函数所属模块、处理的 HTTP 方法（GET/POST/PUT/DELETE）、分类（纯读/纯写/混合/非 JSON）。盘点结果写入 `plans/2026-06-19-API盘点清单.md`，作为后续施加权限和测试的依据
- **Step 2 — 新建 `app_cybersparker/permissions.py`**：`get_role()` / `require_role()` / `deny_user()`
- **Step 3 — 按盘点清单逐个施加权限**：
  - 插件/指纹模块（含调试）→ user 全部 403
  - 其他模块：GET → 不限制；POST/PUT/DELETE → @deny_user
  - 混合方法函数 → 函数内部对写方法做角色检查
  - 非 JSON API → 单独处理
- 执行顺序：插件 → 指纹 → 任务 → 结果 → 配置 → 文件 → AI PoC

### 不做

- 不在 `/files/` 下载端点加角色限制（所有已登录用户均可下载）
- 不改造各模块内部业务逻辑

### 验收条件

- [ ] API 盘点清单已生成，覆盖 `views/expload/` 下全部 28 个 views 文件
- [ ] 根据盘点清单，插件/指纹分类下的全部 API 端点——user 访问均返回 403
- [ ] 根据盘点清单，其他模块分类下的 GET API——user 访问均返回 200（或合理的业务状态码如 404，但不返回 403）
