# 05-代理与运行时配置模块

## 职责

管理 HTTP 代理配置和空间测绘引擎配置，通过全局 Monkey Patch 机制对所有 HTTP 请求注入代理和运行时行为。

## 数据模型

| 模型 | 位置 | 用途 |
|---|---|---|
| `ProxySetting` | `app_cybersparker/models.py:516` | 代理配置（协议、地址、端口） |
| `CyberspaceEngineSetting` | `app_cybersparker/models.py:499` | 空间测绘引擎凭证配置 |

### ProxySetting 约束
- `unique_together(proxy_type, proxy_address, proxy_port)`
- 当前支持 http 和 socks5 协议（`protocol_choices` 含 `(1, "http")` 和 `(4, "socks5")`）

### CyberspaceEngineSetting 约束
- `engine_type` unique
- 支持引擎：fofa / zoomeye / quake / hunter / shodan

## 子模块

### 5.1 代理配置

**关键文件**：`app_cybersparker/views/expload/proxy_setting.py`

**接口**：CRUD (`/proxy_setting` 系列)

**运行时刷新**：
- 应用启动时：`app_cybersparker/apps.py:8` -> `bootstrap_request_runtime()`

- 刷新逻辑（`request_runtime_config_service.py:14`）：读取最新 `ProxySetting`，构建 `http/https` 代理写入 `conf.proxies`

### 5.2 空间测绘引擎配置

**关键文件**：`app_cybersparker/views/expload/cyberspace_engine_setting.py`

**接口**：CRUD (`/cyberspace_engine_setting` 系列)

### 5.3 请求运行时 Monkey Patch

**入口**：`app_cybersparker/services/request_runtime_config_service.py:20`

**Patch 集合**（`app_cybersparker/lib/request_runtime/patch/`）：

| Patch | 文件 | 作用 |
|---|---|---|
| SSL 验证移除 | `remove_ssl_verify.py` | `requests.Session.request` 强制 `verify=False` |
| 警告抑制 | `remove_warnings.py` | 抑制 InsecureRequestWarning |
| URL 解码 | `unquote_request_uri.py` | 对请求 URI 进行 URL 解码 |
| 代理注入 | `hook_request.py` | 注入运行时代理配置 |
| 重定向代理保持 | `hook_request_redirect.py` | 重定向请求保持代理设置 |
| urllib3 解析 Hook | `hook_urllib3_parse_url.py` | 修正 urllib3 URL 解析 |
| HTTP Raw 记录 | `add_httpraw.py` | 请求/响应原始报文记录 |

**初始化流程**：
1. `app_cybersparker/apps.py:8` — AppConfig.ready() -> `bootstrap_request_runtime()`
2. `patch_all_once()` — 对 `requests`/`urllib3` 执行 Monkey Patch
3. `refresh_conf_from_db()` — 从 DB 加载代理配置写入 `conf.proxies`

## 接口

接口详见上方子模块的接口说明。

## 关键约束

- Monkey Patch 为全局生效，所有通过 `requests` 库发出的 HTTP 请求都受影响。
- **任务代理使用 thread-local 存储**（`set_task_proxy` / `_get_task_proxy`，sentinel 模式）：任务线程的代理互不干扰；任务没配代理时强制直连不走 OS 环境变量。
- `conf.proxies` 已废弃（仅保留向后兼容），任务代理不再依赖全局变量。
- SSL 验证全局禁用，仅适合安全测试场景。

## 子模块 5.4：DNSLog 配置

**关键文件**：`app_cybersparker/views/expload/ceye_config.py`

**模型**：`CeyeConfig`（`app_cybersparker/models.py:543`）
- `api_token`：ceye.io API Token
- `identifier`：ceye.io 分配的唯一域名标识符
- 全局单条记录，CRUD 页面参照 proxy_setting 模式

**用途**：Nuclei 模板执行时注入 `{{ceye_url}}` 和 `{{interactsh-url}}` 动态变量（两者等价，都映射到 ceye），配合 ceye API 轮询实现 DNS 外带漏洞验证。

**当前限制**：仅支持 ceye DNS 日志类型（`type=dns`），暂不支持 HTTP 请求日志（`type=request`）。Interactsh 协议不支持，`{{interactsh-url}}` 变量被当作 `{{ceye_url}}` 降级处理。
