# 08-前后端分离模块

> 最后更新：2026-06-01（反映架构收口完成状态）

## 职责

在不动现有 Django 扫描、调度、写库主链路的前提下，把后台页面层逐步改成 React 前端 + Django API 的结构。**已全部完成：20 个 React 页面、40+ 个 `/api/v1` 接口。**

## 实现状态

已完成。全部 50 项 backlog 已验收，92 项标记为已验收。侧边栏导航、路由、鉴权全部由 React 接管。旧 Django 后台模板已删除，仅保留 `login.html`；回退不再依赖旧模板，而是依赖 React 壳页兜底。

## 技术基线（实际）

- 前端：React 18 + Vite + TypeScript + antd 6.x
- 路由：react-router-dom BrowserRouter（basename=/react-shell）
- 登录态：同域部署，Django session cookie
- 鉴权：AuthGuard 组件 + Django AuthMiddleware + BasicAuthMiddleware
- API：`/api/v1` 统一分页、错误码、CSRF 规则
- 静态资源：`app_cybersparker/static/react-shell/assets/`（Vite 自动 chunk）
- 壳页：生产环境由 Nginx 直接提供 Vite 构建的 `index.html`，开发环境 Django catch-all 兜底
- Django 中间件放行：`/react-shell/*`、`/api/v1/*`（未登录返 401 JSON）、`/static/*`


### 分页与鉴权

- 列表接口统一使用 `page`、`rows_per_page`
- 返回结构：`items`、`page`、`rows_per_page`、`total`、`total_pages`
- GET 只读接口不要求前端额外处理 CSRF
- POST/PUT/DELETE 统一要求同源 cookie + `X-CSRFToken`
- 未登录访问 `/api/v1/*`（除 auth/session）返回 401 JSON：`{"code":"UNAUTHENTICATED","message":"login required","login_url":"/login"}`

### 批量操作

以下模块提供 `POST /api/v1/<module>/batch-delete` 批量删除端点（共 12 个）：

| 模块 | 路由 |
|------|------|
| plugins / dicts / dict-groups / proxies / cyberspace-engines / fingerprints | `/api/v1/<module>/batch-delete` |
| exp-results / auto-exp-results / identify-tasks / batch-tasks / dirscan-tasks / export-tasks | `/api/v1/<module>/batch-delete` |

所有 batch-delete 使用 `POST` 方法，body 传 `{"ids": [...]}`。

## 页面路由

生产环境由 Nginx 直接提供 React 编译产物（`deploy/nginx/react-shell.conf`），Django 不参与页面渲染。每个 React 页面对应一个 Nginx 静态文件路径，通过 `try_files $uri /react-shell/index.html` 保证刷新不 404。

Django 侧的 URL 路由（`/react-shell/*`）保留作为回退（catch-all 兜底到 react_shell 视图）：

- 显式注册：`react-shell/`、`plugins`、`dicts`、`dict-groups`、`proxies`、`engines`、`fingerprints`、`identify-tasks`、`batch-tasks`、`dirscan-tasks`、`auto-exp-results`、`exp-results`、`export-tasks`、`ceye-config`、`dashboard`、`exp-debug`、`fingerprint-debug`、`assets/search`
- catch-all：`react-shell/<path:subpath>` → 兜底所有未显式注册的 React 子路径

> **有意豁免**：`/react-shell/ip-detail` 和 `/react-shell/plugins/:id` 未注册显式 Django 壳页——ip-detail 在新标签页打开无需侧边栏；plugins/:id 是 plugins 的子路由，由 React Router 接管。两者通过 catch-all 兜底，刷新不 404。

## 关键约束

1. Django 继续负责登录、鉴权、数据库、Celery、Redis、扫描执行、结果写库。
2. React 接手所有后台内容区页面和侧边栏导航。
3. 旧 Django 模板（27 个）已于 2026-06-03 删除，仅保留 `login.html`。回退路径改为 Django catch-all 兜底到 React 壳页。
4. `/api/v1` 接口是对旧 Django 视图的契约化包装，不改变底层扫描/写库语义。
5. 所有页面支持浅色/深色主题切换（localStorage 持久化）。
6. Vite 自动代码拆分，按需加载。
