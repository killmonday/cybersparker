# 09-文件管理模块

## 职责

管理用户上传的目标文件（`EXP_input/` 目录），提供独立的上传/下载/删除页面；测绘引擎数据文件（`EXP_input/engine_assets/`）自动定期清理。

## 数据存储

- 文件元信息**不建数据库模型**，直接从文件系统读取（文件名、大小、mtime、行数）
- 文件存储在 `EXP_input/`（项目根目录下，与现有任务表单的"历史上传文件"共用同一目录）
- 测绘引擎数据文件存储在 `EXP_input/engine_assets/`

## 页面

| 页面 | React 路由 | 说明 |
|------|-----------|------|
| 文件管理 | `/react-shell/target-files` | 文件列表、上传、下载、删除 |

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/target-files` | 列出 `EXP_input/` 下所有 .txt 文件（文件名、大小、行数、mtime），排除子目录 |
| POST | `/api/v1/target-files/upload` | 上传 .txt 文件（限 10MB），重名自动加后缀；前端+后端双重校验后缀 |
| GET | `/api/v1/target-files/<filename>/download` | 下载指定文件 |
| DELETE | `/api/v1/target-files/<filename>` | 删除文件，返回被引用任务列表（如有）；确认后同步清理引用任务的 `history_files` 字段 |
| POST | `/api/v1/target-files/batch-delete` | 批量删除，逐文件返回引用检查结果 |

### 安全约束

- 所有接受文件名的 API 必须做**双重路径校验**：
  1. `get_absolute_target_path()` 检查不逃逸项目根目录
  2. 额外检查解析路径在 `EXP_input/` 目录内且不在任何子目录下（防御 `../` 穿越到 `settings.py` 等文件）
- 所有 API 沿用现有 AuthMiddleware 会话认证
- 上传大小限制 30MB

## 定时任务

| 任务 | 队列 | 频率 | 说明 |
|------|------|------|------|
| 清理过期引擎数据 | `maintenance` | 每天 | 删除 `EXP_input/engine_assets/` 下 mtime > 60 天的文件 |

Celery Beat 调度配置需新增到 `settings.py`（`CELERY_BEAT_SCHEDULE`）。

## 与其他模块的关系

- 任务执行模块（02）的"历史上传文件"选择器读同一目录，无需对接
- 删除文件时检查并同步清理 `auto_scan_tasks.history_files` 和 `batch_EXPTask.history_files` 字段中的引用
- 现有 `GET /api/v1/history-files` 和 `GET /api/v1/identify-tasks/history-files` 保持不变（向后兼容），新 API `GET /api/v1/target-files` 额外返回行数

## 约束

- 仅允许 `.txt` 后缀
- 重名不覆盖，自动加数字后缀（并发安全：循环检查文件系统）
- 不管理子目录文件（`.merged/`、`engine_assets/`）
- 上传大小限制 30MB
- 不校验文件内容格式
