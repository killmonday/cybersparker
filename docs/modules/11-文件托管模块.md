# 11-文件托管模块

> 最后更新：2026-06-19 | 状态：已实现

## 概述

提供一个通用的文件托管 Web 服务。用户在后台上传任意文件，可设置公开（公网任意下载）或需登录鉴权，生成直接下载链接。

与现有"文件管理"（模块 09）的区别：文件管理面向任务输入文件（仅 .txt），文件托管面向任意文件的公网分发。

## 数据模型

### HostedFile

| 字段 | 类型 | 说明 |
|------|------|------|
| id | AutoField | 主键 |
| original_name | CharField(512) | 用户上传时的原始文件名 |
| stored_name | CharField(512) | 磁盘上的实际文件名（上传时生成，避免冲突） |
| file_size | BigIntegerField | 文件大小（字节） |
| note | TextField | 备注（可空，最多500字） |
| is_public | BooleanField | True=公开（无需登录可下载），False=需登录 |
| created_at | DateTimeField | 上传时间 |
| updated_at | DateTimeField | 最后修改时间 |

## API

### 后台管理 API（需登录，`/api/v1/hosted-files`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/hosted-files` | 文件列表 |
| POST | `/api/v1/hosted-files/upload` | 上传文件（multipart，字段：file + is_public） |
| DELETE | `/api/v1/hosted-files/<id>` | 删除文件（同时删磁盘文件） |
| PUT | `/api/v1/hosted-files/<id>/rename` | 重命名（JSON body: new_name） |
| PUT | `/api/v1/hosted-files/<id>/note` | 编辑备注（JSON body: note） |

### 公开下载入口（无需登录，`/files/<id>/<原始文件名>`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/files/<id>/<原始文件名>` | 公开文件直接下载；鉴权文件检查登录状态，未登录返回 403 |

## 存储

- 目录：项目根目录 `./upload_files/`
- 单文件上限：200MB
- 无文件类型限制
- 上传时生成唯一磁盘文件名避免冲突

## 前端

- 页面路由：`/react-shell/hosted-files`
- 侧边栏：独立一级菜单"文件托管"
- 使用 antd 6.x 组件（Table/Upload/Modal/Button/Select/Tag）
