# 12-用户管理模块

> 最后更新：2026-06-19 | 状态：已实现

## 概述

给后台新增完整的用户管理体系。使用 Django 内置 `auth.User` 存储账户，新增 `UserProfile`（OneToOne）存储角色。角色分三层：超级管理员、普通管理员、普通用户。权限在后端 API 层执行。

## 数据模型

### UserProfile

| 字段 | 类型 | 说明 |
|------|------|------|
| id | AutoField | 主键 |
| user | OneToOneField(auth.User) | 关联 Django User |
| role | CharField(16) | super_admin / admin / user |

`auth.User` 自带 username、password（PBKDF2 哈希）、is_active、date_joined、last_login。

## API

全部在 `/api/v1/users` 下，需登录。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/users` | 用户列表（super_admin 看全部，admin 仅看 user，user 不可访问） |
| POST | `/api/v1/users` | 创建用户（JSON body: username, password, role） |
| DELETE | `/api/v1/users/<id>` | 删除用户（同时删除所有 session） |
| PUT | `/api/v1/users/<id>/role` | 修改角色（仅 super_admin，改后清除 session） |
| PUT | `/api/v1/users/<id>/password` | 修改他人密码（super_admin 任何人，admin 仅 user） |
| PUT | `/api/v1/users/me/password` | 修改自己密码 |

## 角色权限

| 角色 | 用户管理 | 功能模块写操作 | 插件/指纹查看 | 其他模块查看 |
|------|:---:|:---:|:---:|:---:|
| super_admin | 管所有人 | ✓ | ✓ | ✓ |
| admin | 仅管 user | ✓ | ✓ | ✓ |
| user | ✗ | ✗ | ✗ | ✓ |

## 权限执行

- 后端：`app_cybersparker/permissions.py`（`require_role` / `deny_user` 装饰器 + `get_role` 工具函数）
- 前端：各页面按钮按 `role !== 'user'` 条件渲染；侧边栏不过滤、不加路由守卫
- 特殊：`/files/` 下载入口不做角色限制，所有已登录用户均可下载

## 前端

- 用户管理页面：`UserManagementPage.tsx`，路由 `/react-shell/users`
- AuthContext：`frontend/src/contexts/AuthContext.tsx`（全新），持有 `{ username, role }`
- 各已有页面：按钮用 `useAuth().role !== 'user'` 条件渲染

## 初始化

Migration 中创建超级管理员 `admin` / `admin`，role=super_admin。
