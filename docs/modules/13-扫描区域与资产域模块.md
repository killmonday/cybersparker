# 13-扫描区域与资产域模块

> 最后更新：2026-06-21 | 状态：已实现，已验收

## 概述

给系统引入“扫描区域（AssetZone）”这一层归属信息，让公网资产和多个内网资产域可以在同一套任务、检索、目录扫描、漏洞验证流程中并存，不再因为相同的 `ip:port` 互相覆盖或串线。

本模块不是单独的扫描器，而是一个跨模块的基础能力：

- 自动扫描任务写资产时，必须知道资产属于哪个区域
- 批量任务、目录扫描任务从资产表读输入源时，必须知道自己只读哪个区域
- 全局资产检索既要能看全部区域，也要能按区域过滤
- 目录扫描结果必须能绑定回“哪一条根资产”，不能只靠 `protocol + host + port` 模糊识别

## 数据模型

| 模型 | 位置 | 用途 |
|------|------|------|
| `AssetZone` | `app_cybersparker/models.py`（新增） | 扫描区域定义，如“公网”“内网1”“客户A办公网” |
| `auto_scan_indentify_result` | `app_cybersparker/models.py` | 识别结果主表，新增 `zone` FK，唯一键改为 `(zone, protocol, host, port, uri_path)` |
| `auto_scan_tasks` | `app_cybersparker/models.py` | 自动扫描任务，新增 `zone` FK；fscanx 任务也复用此模型 |
| `batch_EXPTask` | `app_cybersparker/models.py` | 批量任务，新增 `zone` FK |
| `DirScanTask` | `app_cybersparker/models.py` | 目录扫描任务，新增 `zone` FK |
| `auto_scan_directory_result` | `app_cybersparker/models.py` | 目录扫描结果，新增 `zone` FK 和 `root_identify_result` FK |
| `AssetRootBinding` | `app_cybersparker/models.py`（新增） | 固定某个 `zone + protocol + host + port` 对应的根资产是谁，避免目录扫描根资产身份漂移 |

### AssetZone 约束

- 固定存在一条系统区域：`code="public"`，`name="公网"`
- 公网区域不可删除
- 区域名称全局唯一
- 删除区域前必须确认：
  - 该区域下没有资产
  - 该区域下没有自动扫描任务
  - 该区域下没有批量任务
  - 该区域下没有目录扫描任务
  - 该区域下没有目录扫描结果

## 核心规则

### 1. 资产归属

`auto_scan_indentify_result` 的唯一键从“全局唯一”改为“区域内唯一”：

- 旧：`(protocol, host, port, uri_path)`
- 新：`(zone, protocol, host, port, uri_path)`

这意味着：

- 公网里的 `1.2.3.4:80` 仍然只有一条资产
- “内网1”的 `192.168.1.1:80` 和“内网2”的 `192.168.1.1:80` 可以同时存在

### 2. 任务归属

以下任务都必须显式属于一个区域：

- 自动扫描任务
- fscanx 导入任务（本质是 `auto_scan_tasks.input_type=2`）
- 批量任务
- 目录扫描任务

测绘引擎相关输入源（实时测绘 / 历史测绘结果）固定属于公网，不允许保存成内网区域。

### 3. 输入源读取规则

凡是“从资产表读输入源”的地方，都必须强制限定到任务自己的区域：

- 自动扫描任务：从检索语句导入
- 批量任务：从检索语句导入
- 目录扫描任务：手动选择任务、全选所有任务、从检索语句导入

批量任务的 `history vuln assets` 是本模块里的特殊链路：

- 本期只支持自动扫描链路产生的 `auto_scan_exp_result`
- 优先通过 `identify_result_id -> asset.zone` 过滤
- 兼容旧自动扫描漏洞结果时，才允许 `target + task.zone` 唯一反查
- 不把 `EXPTask_result` 纳入本期 zone 化范围

### 4. 全局检索语义

全局资产检索的默认语义是：

- 默认跨全部区域检索
- 提供可选区域过滤下拉（含"所有内网"聚合选项，后端以 `exclude(zone__code='public')` 匹配所有非公网 zone）
- 详情、facet、端口概览等附属接口都跟随当前区域过滤条件

### 5. 目录扫描根资产归属

目录扫描不能再只靠 `(protocol, host, port)` 识别根资产。

新规则：

- 新增 `AssetRootBinding`，固定某个 `zone + protocol + host + port` 对应的根资产 id
- 构建目录扫描快照时，冻结根资产 `identify_result_id`
- 目录扫描结果表保存 `root_identify_result`
- 目录扫描结果按资产维度共享快照存储：同 zone 下同一台资产同一路径只保留一份当前结果，后跑任务允许刷新前一条结果
- 根路径回写按 `root_identify_result_id` 定位，不再按三元组模糊更新
- 被 `AssetRootBinding`、目录扫描结果或漏洞结果引用的资产进入“保留锚点资产”集合，不允许被普通孤立资产清理删除

## API（规划）

### 扫描区域管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/zones` | 区域列表 |
| POST | `/api/v1/zones` | 新增区域 |
| PUT | `/api/v1/zones/<id>` | 改名/改备注 |
| DELETE | `/api/v1/zones/<id>` | 删除区域（有引用时拒绝） |

### 任务表单影响

以下任务相关 API 的请求/响应都要带 `zone_id`：

- `/api/v1/identify-tasks`
- `/api/v1/batch-tasks`
- `/api/v1/dirscan-tasks`
- `/api/v1/fscanx-tasks`

### 检索页影响

- `/api/v1/assets/search`：新增可选 `zone_id`
- facet / 详情 / 端口概览等附属接口：统一接受并沿用 `zone_id`

## 前端

### 新页面

- 扫描区域管理页：`/react-shell/zones`
- 侧边栏菜单位置：任务管理下新增“扫描区域”

### 现有页面变化

- 自动扫描任务页：新增 zone 选择
- 批量任务页：新增 zone 选择
- 目录扫描任务页：新增 zone 选择
- fscanx 任务页：新增 zone 选择
- 全局资产检索页：新增 zone 过滤下拉 + 结果表显示区域列
- 任务列表/详情页：显示任务所属区域

## 风险点

1. 区域归属是跨模块基础能力，遗漏任何一条读/写链路都会出现“表面支持 zone，底层仍串线”的问题
2. 目录扫描是本模块里风险最高的部分，因为它原本依赖 `(protocol, host, port)` 来定位根资产
3. 结果展示页的附属接口（facet、详情、端口概览）如果不统一带 zone，会出现主列表已过滤但详情跨区串出的情况
