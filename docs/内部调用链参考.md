## 4) 自动识别与联动漏洞扫描链路（Identify_task）

### 流程总览

```
用户上传目标文件 → HTTP请求采集 → 指纹识别(Identifyner) → 入库(auto_scan_indentify_result)
    → [可选] POC关联(get_exp_ids_for_products) → 动态加载POC(poc_runtime_resolver)
    → 执行verify → 入库(auto_scan_exp_result)
```

四队列生产者-消费者流水线：

```
[目标文件] → queue_input → [HTTP请求线程×N] → queue_fingerpoint_input
    → [指纹识别线程×N] → queue_EXP_input → [漏洞扫描线程×N]
    → queue_EXP_result → [结果入库线程]
```

### 详细时序

```mermaid
sequenceDiagram
    actor User as 用户
    participant View as auto_scan_task.py
    participant DB as Django ORM
    participant Handler as Auto_exploit_Task_handler
    participant Producer as Producer线程
    participant ReqCon as RequestConsumer×N
    participant FP as Identifyner
    participant FPCon as FingerprintConsumer×N
    participant ExpCon as EXPConsumer×N
    participant Resolver as poc_runtime_resolver
    participant SaveRes as SaveResult线程

    Note over User,SaveRes: === 阶段0: 任务创建 ===
    User->>View: POST /Identify_task/add (上传URL文件 + Vulnerability_scanning=1)
    View->>DB: INSERT auto_scan_tasks (status=3/stop)
    DB-->>View: OK

    Note over User,SaveRes: === 阶段1: 启动 ===
    User->>View: POST /Identify_task/operate (status=0)
    View->>DB: UPDATE status=2(running)
    View->>Handler: startTask→Thread(args=row_dict).start()

    Note over Handler: === 阶段2: 流水线初始化 ===
    Handler->>Handler: 创建4队列 + FP.Identifyner()加载全部指纹ID
    Handler->>Handler: 启动 Producer + SaveResult + N×(ReqConsumer + FPConsumer + [ExpConsumer])

    Note over Producer,ReqCon: === 阶段3: 读取目标 ===
    loop 逐行读取目标文件 (断点续读 current_line)
        Producer->>ReqCon: queue_input.put(url)
    end

    Note over ReqCon,FP: === 阶段4: HTTP采集 ===
    ReqCon->>ReqCon: requests.get(url)→header/body/title/status_code
    ReqCon->>FPCon: queue_fingerpoint_input.put({url: {header,body,title,status_code}})

    Note over FP,DB: === 阶段5: 指纹识别 ===
    FPCon->>FP: Identifyner.handle(header, body, title)
    loop 遍历 fingerPrint 全部 ID
        FP->>DB: SELECT product, condition FROM fingerPrint WHERE id=?
        DB-->>FP: product + condition
        alt 精确匹配: title="xxx"/body="xxx"/header="xxx"
            FP->>FP: 子串包含匹配
        else 正则匹配: title~="re"/body~="re"/header~="re"
            FP->>FP: regulations_mate() 正则匹配
        else 布尔组合: || / && / 括号嵌套
            FP->>FP: 递归解析复合表达式
        end
    end
    FP-->>FPCon: fingers_list = ["Apache","PHP",...]
    FPCon->>DB: INSERT auto_scan_indentify_result
    alt Vulnerability_scanning == 1
        FPCon->>ExpCon: queue_EXP_input.put({url: fingers_list})
    end

    Note over ExpCon,SaveRes: === 阶段6: POC关联与扫描 ===
    ExpCon->>DB: get_exp_ids_for_products(fingers_list)<br/>fingerPrint→exp_relate_fingerprint→EXP (三层JOIN)
    DB-->>ExpCon: {exp_id: {poc, product}}
    loop 每个关联EXP
        ExpCon->>Resolver: load_runtime_module_from_poc(poc_path, exp_id)
        alt .py POC
            Resolver->>Resolver: importlib动态加载Python模块
        else .yaml/.yml (Nuclei模板)
            Resolver->>Resolver: compile→Python wrapper(内嵌YAML hex)<br/>→nuclei_runtime_engine解析YAML requests/matchers
        end
        Resolver-->>ExpCon: exp_module
        ExpCon->>Resolver: call_runtime_method(exp, "verify", url)
        Resolver-->>ExpCon: {target, result, exp_id, product}
        ExpCon->>SaveRes: queue_EXP_result.put(result)
    end

    Note over SaveRes: === 阶段7: 落库 ===
    SaveRes->>DB: INSERT auto_scan_exp_result

    Note over Handler: === 阶段8: 完成 ===
    Producer->>DB: UPDATE status=1(finish), endTime=now, process=100%
```

### 指纹规则语法

| 语法 | 含义 | 示例 |
|------|------|------|
| `title="xxx"` | 标题包含 | `title="Apache Tomcat"` |
| `body="xxx"` | 响应体包含 | `body="wp-content"` |
| `header="xxx"` | 响应头包含(默认) | `header="nginx"` |
| `title~="re"` | 标题正则 | `title~="Apache/[\d.]+"` |
| `body~="re"` | 响应体正则 | `body~="phpmyadmin"` |
| `header~="re"` | 响应头正则 | `header~="Server: .*"` |

布尔逻辑：`||`(或)、`&&`(且)、括号嵌套如 `title="php"||(body="mysql"&&header="apache")`

### POC运行时解析 (poc_runtime_resolver)

| POC类型 | 加载方式 |
|---------|----------|
| `.py` (Python3) | `importlib.util.spec_from_file_location` 动态加载为模块 |
| `.yaml/.yml` (Nuclei) | 直接走 `poc_runtime_resolver._build_yaml_wrapper()` → `nuclei_runtime_engine.run_nuclei_template()`，不再生成中间 `.py` 文件 |

统一调用入口：`call_runtime_method(exp, "verify", url)`，nuclei_yaml 仅支持 verify 模式。

当前协议边界：
- 支持：`http/requests`、`tcp/network`
- 不支持：`code/javascript/headless/file/dns/ssl/websocket/whois`
- 导入命令会直接跳过不支持协议模板；历史数据统一通过 `cleanup_nuclei_unsupported_templates` 清理。

### EXP-指纹关联查询

```
识别产品列表 → fingerPrint 表(按product查) → exp_relate_fingerprint 关联表 → EXP 表
```

同一产品可关联多个 EXP，按产品名字典序取优先级最高者。
