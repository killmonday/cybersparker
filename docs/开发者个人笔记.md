# MAX_EXPLOIT_THREAD_NUM
MAX_EXPLOIT_THREAD_NUM = 50 影响全部三种任务：

如果 MAX_EXPLOIT_THREAD_NUM = 50
        │
        ├── 自动扫描任务
        │   ├── 表单校验：thread_num ≤ MAX_EXPLOIT_THREAD_NUM
        │   ├── HTTP 并发：network_concurrency = min(thread_num, MAX_EXPLOIT_THREAD_NUM)
        │   ├── 指纹识别线程：min(3, thread_num, MAX_EXPLOIT_THREAD_NUM, lease)
        │   └── 漏洞扫描线程：min(thread_num, MAX_EXPLOIT_THREAD_NUM, lease-指纹)
        │
        ├── 批量任务
        │   ├── 表单校验：thread_num ≤ MAX_EXPLOIT_THREAD_NUM
        │   ├── 线程模式：consumer_exp 数量 = min(thread_num, MAX_EXPLOIT_THREAD_NUM, lease)
        │   └── 协程模式：不受此限制（走 GLOBAL_COROUTINE_LIMIT）

如果你想放开自动扫描的漏洞线程数，MAX_EXPLOIT_THREAD_NUM 调大就行。 指纹识别已经硬编码上限 3，不受这个值影响。其余环节会在 用户在表单设置的线程数 和全局 GLOBAL_THREAD_LIMIT 之间取最小值。
