# 数据库恢复指南

当 PostgreSQL 业务数据丢失时，按以下步骤恢复。

## 前置条件

以下文件必须存在：

- `/workspaces/cybersparker/db/exp.db` — 历史指纹库（SQLite，5,897 条）
- `/workspaces/cybersparker/fingerprint.txt` — 指纹规则文件（31,957 行）
- `/tmp/nuclei-templates` — Nuclei 官方模板仓库
- `python manage.py` 可正常运行，数据库连接正常

## 恢复步骤

### 1. 清空

```bash
python manage.py shell -c "
from app_cybersparker.models import fingerPrint, EXP, cveExtensions, exp_relate_fingerprint
exp_relate_fingerprint.objects.all().delete()
cveExtensions.objects.all().delete()
EXP.objects.all().delete()
fingerPrint.objects.all().delete()
"
```

### 2. 从 exp.db 导入历史指纹

```bash
python import_fingerprint_from_exp_db.py
```

约 5,897 条，保留原始创建时间。

### 3. 从 fingerprint.txt 导入指纹规则

```bash
python manage.py import_fingerprints
```

约 28,700 条（与步骤 2 的 5,897 条自动去重合并），最终约 34,000 条。

### 4. 从 nuclei-templates 导入 EXP

```bash
python manage.py import_nuclei_templates --source /tmp/nuclei-templates
```

只会导入当前引擎支持的模板（`http/requests`、`tcp/network`）。带 `code/javascript/headless/file/dns/ssl/websocket/whois` 顶层协议块的模板会被直接跳过，不再入库。

清理历史不支持模板：

```bash
python manage.py cleanup_nuclei_unsupported_templates
```

## 验证

```bash
python manage.py shell -c "
from app_cybersparker.models import fingerPrint, EXP, cveExtensions, exp_relate_fingerprint
print(f'指纹: {fingerPrint.objects.count()}')
print(f'EXP:  {EXP.objects.count()}')
print(f'cveExtensions: {cveExtensions.objects.count()}')
print(f'指纹绑定: {exp_relate_fingerprint.objects.count()}')
"
```

预期：指纹 ≈34,000，EXP ≈ 11,109，cveExtensions ≈ 11,109，指纹绑定数量低于旧口径（因为不支持协议模板不会再入库）。

## 注意事项

- 步骤 3 和 4 需要较长时间（指纹约 1 分钟，EXP 约 5 分钟），不要中断。
- 指纹导入自动去重（按 condition 字段），重复导入不会产生重复记录。
- 如果 `/tmp/nuclei-templates` 不存在，需先从 `projectdiscovery/nuclei-templates` 拉取。
- exp.db 仅包含 78 条 EXP 记录（Python 插件），如需恢复这些，可单独从 exp.db 导入 EXP 表。
