# 需求规格说明书初稿

> **历史参考**：本文档写于项目早期（阶段一），内容已严重滞后于当前实现。接口路径、模块划分、架构描述的当前状态以 `docs/当前实现总览.md` 和 `docs/项目控制台.md` 为准。本文档保留作为初始设计意图的记录。

## 系统定位

漏洞利用平台（cybersparker）是一个面向安全测试人员的漏洞验证与批量利用系统。支持通过 Python 插件和 Nuclei YAML 模板执行漏洞检测，提供单任务、批量任务、自动识别三种执行模式。

## 核心用户角色

- **安全测试人员**：创建插件、执行任务、查看/导出结果。
- **平台管理员**：管理代理配置、空间测绘引擎配置。

## 功能需求

### FR-1 插件管理
- 支持上传/在线编辑 Python3 插件和 Nuclei YAML 模板。
- 支持插件启停控制（`use` 字段）。
- 支持插件调试执行（`/expload/debugging` -> 即时执行并查看输出）。
- 插件类型包括命令执行、代码执行、SQL 注入、信息泄露、文件上传、文件读取、目录遍历、CSRF、身份绕过、弱口令、路径泄露等。

### FR-2 单任务执行
- 基于单个插件 + 目标文件（或历史结果）创建执行任务。
- 生产者-消费者线程模型，可配置线程数和请求间隔。
- 任务状态：finish / running / stop。
- 支持任务启停控制，运行时句柄存入 `THREAD_DIC`。

### FR-3 批量任务执行
- 多插件、多输入源批量执行。
- 输入源类型：文件上传、历史漏洞资产、历史上传文件、网络空间测绘引擎。
- 运行模式：线程模式 / 协程模式（gevent 子进程）。
- 支持多空间测绘引擎（fofa / zoomeye / quake / hunter / shodan）。
- 任务状态：finish / running / stop。

### FR-4 自动识别任务
- 基于指纹规则库对目标进行产品识别。
- 可选联动漏洞扫描（识别到产品后自动调用对应插件）。
- 识别结果：产品、目标、IP、端口、协议、国家、地区、状态码、响应头。
- 任务状态：finish / running / stop。

### FR-5 结果管理
- 统一结果列表，按 task_type 区分单任务/批量任务结果。
- 结果操作：查看详情、漏洞复验、下载导出、删除。
- 自动识别结果和联动漏洞结果独立展示和导出。

### FR-6 代理与运行时配置
- 代理配置 CRUD（协议、地址、端口）。
- 应用启动时自动加载代理配置到运行时。
- 代理增删改时自动刷新运行时。
- 每个请求自动使用运行时代理（通过 monkey patch）。

### FR-7 空间测绘引擎配置
- 多引擎配置管理（fofa / zoomeye / quake / hunter / shodan）。
- 引擎 API 地址、认证信息、代理关联。

### FR-8 认证与访问控制
- 登录认证（Django session + Basic Auth）。
- 全量 URL 认证中间件（排除 `/login`）。
- Basic Auth 中间件作为补充认证层。

### FR-9 指纹管理
- 指纹规则 CRUD（产品名称 + 匹配条件）。
- 指纹调试执行（验证规则匹配效果）。
- 插件与指纹关联（多对多，`exp_relate_fingerprint`）。

### FR-10 插件运行时引擎
- Python3 插件：`importlib` 动态加载 + `reload`。
- Nuclei YAML 模板：解析 YAML 并转换为统一执行模型。
- 插件约定函数：`_verify` / `_cmd_exc` / `_code_exc` / `_file_read` / `_attact`。
