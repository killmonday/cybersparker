# 项目风险项

> 最后更新：2026-06-07

## RISK-002 全局资产检索 facet 聚合在超大数据量下可能较重

- 等级：低
- 位置：`app_cybersparker/views/expload/task_manage/auto_scan_result.py`
- 现象：全局资产检索的 favicon、证书、C 段等 facet 会在用户展开聚类时做分组统计和计数。
- 场景：如果资产表增长到几十万行，用户用很宽泛的条件搜索后再展开聚类，数据库需要对大量资产做分组统计，页面可能等待更久。
- 当前处理：保留现有懒加载策略，不立即加缓存或索引；等真实慢查询日志或用户反馈出现后，再按具体字段优化。
- 观察信号：全局资产检索接口响应明显变慢，或数据库慢查询集中出现在 facet 分组统计语句。

## 暂不作为风险项的问题

- 历史测绘结果列表逐文件统计目标数已在 2026-06-07 改为块读取统计非空行，用来延缓大量历史任务时的加载问题。
