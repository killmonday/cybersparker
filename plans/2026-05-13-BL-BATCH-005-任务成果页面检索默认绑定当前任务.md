# BL-BATCH-005: 任务成果页面检索默认绑定当前任务

- 日期：2026-05-13
- 状态：已完成

## 问题

`/batch_exploadTask/<uid>/result` 页面的检索按钮，不输入任何检索条件时点击搜索，展示了所有任务的成果，而非当前任务的成果。

## 根因分析

`batch_exp_task.py:task_result` 中，搜索表单的 `plugin_name` select 默认值为 "all"。当用户未输入任何搜索条件直接点击检索按钮时，表单提交 `plugin_name=all`，使 `search_dict` 为 `{"plugin_name": "all"}`（truthy）。后端进入 `if search_dict:` 分支执行查询：

```python
models.EXPTask_result.objects.filter(_query, task_type=2)  # 缺少 task_id=uid
```

而 `plugin_name=all` 在构建 `_query` 时被 `pass` 跳过，`_query` 为空 Q()，等效于查询所有 task_type=2 的结果。

## 其他页面排查

| 页面/URL | 视图 | 结果 |
|-----------|------|------|
| `/batch_exploadTask/<uid>/result` | `batch_exp_task.task_result` | **BUG** — search_dict 分支缺少 task_id |
| `/exploadTask/<uid>/result` | `exp_task.Task_result` | OK — 使用 q 参数，始终 scope task_id |
| `/identify_exp_result/<uid>/list` | `auto_scan_exp_Result.list` | OK — 使用 q 参数，始终 scope task_id |
| `/Identify_task_Result/<uid>/result` | `auto_scan_result.Task_result` | OK — 始终 scope task_id |
| `/result_List` | `expResult.result_list` | N/A — 全局页面，all() 预期行为 |
| `/auto_exp_result` | `all_auto_exp_result.list` | N/A — 全局页面 |
| `/all_Indentify_result` | `all_Indentify_result.list` | N/A — 全局页面 |

## 修复方案

`batch_exp_task.py:task_result` 的 `if search_dict:` 分支添加 `task_id=uid` 条件：

```python
# 修复前
queryset = models.EXPTask_result.objects.filter(_query, task_type=2).values(...)

# 修复后
queryset = models.EXPTask_result.objects.filter(_query, task_id=uid, task_type=2).values(...)
```

## 风险

极低 — 仅在批量任务成果页面添加已有的 task_id 过滤条件，不改变其他行为。

## 验证

- Python 语法检查通过
- 修复后 `if search_dict:` 和 `else` 分支均包含 `task_id=uid`，无论是否输入搜索条件，结果始终限定在当前任务
