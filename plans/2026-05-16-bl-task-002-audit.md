# BL-TASK-002 单任务 exploadTask 去留审计

## 审计数据

| 维度 | 发现 |
|------|------|
| `EXPTask.taskType` | 2 种：1=Verify（验证漏洞），2=Attact（执行攻击命令） |
| `cmd_input` | 仅 taskType=2 时生效，用户输入的 Shell 命令字符串 |
| 前端入口 | `exp_task_list.html`、`exp_task_add.html`、`exp_task_edit.html`、`exp_task_detail.html` |
| API | `GET/POST /exploadTask/list`、`/exploadTask/add`、`/exploadTask/edit`、`/exploadTask/detail`、`/exploadTask/delete`、`/exploadTask/operate` |
| 结果页 | `GET /exploadTask/<id>/result`，过滤 `task_type=1` |
| 执行器 | `single_task_executor.py:184` — taskType=1 调 `verify`，taskType=2 调 `attact` + `cmd_input` |
| 结果落库 | 直接写 `EXPTask_result`（`task_type=1`），不走事件缓冲 |
| 启停 | `THREAD_DIC` 内存字典 |

## 批量任务对比

| 维度 | 单任务 | 批量任务 |
|------|--------|----------|
| Verify | 支持 | 支持 |
| Attact + cmd_input | 支持 | **不支持** |
| 多插件 | 单插件 | 多插件（逗号分隔） |
| Celery 调度 | 无 | 已迁移（BL-BATCH-007） |
| 资源预算 | 无 | 已接入（BL-BATCH-008） |
| 结果缓冲 | 直接写 DB | Redis Stream + DB writer |
| Stop bridge | 无（THREAD_DIC） | Redis/DB stop signal |
| gevent 模式 | 无 | 支持 |

## 结论

**建议：迁移到批量任务（保留 Attact 能力）。**

Attact/cmd_input 是单任务对批量任务的唯一功能优势。`poc_runtime_resolver.py` 已验证支持 `_attact` / `_attack` 方法解析。建议后续增加一个 backlog item 在批量任务中补齐 `task_type` + `cmd_input` 支持，完成后将单任务 UI 入口设为只读归档。

## 后续开发事项

- 批量任务新增 `task_type`（Verify/Attact）和 `cmd_input` 字段
- 批量任务消费者（`batch_task_executor.py:consumer_exp`）按 `task_type` 选择 `verify` 或 `attact` + `cmd_input`
- 单任务前端入口、API 标记为 deprecated 或移除
