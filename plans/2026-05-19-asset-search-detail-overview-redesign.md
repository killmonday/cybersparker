# 资产检索页详情区大优化

- 状态：已完成
- 创建时间：2026-05-19
- 关联 Backlog：BL-AUTO-008 / BL-AUTO-009

## 做什么

重构 standalone 结果页 / 全局资产检索页的资产详情展示区：
1. 删除重复的“识别产品”“开放端口”块。
2. 将“端口-产品-协议”改为“端口概览”，第一行突出当前端口，展示协议、端口、复制 URL、跳转 URL、识别产品。
3. 其他端口排在下方，每行固定宽度，仅产品区域支持横向滚动，滚动条改浅色半透明。
4. 在端口概览右侧新增“受影响漏洞”区域，展示漏洞插件名、CVE 编号、查看结果按钮。
5. 点击“查看结果”后弹窗查看保存到数据库的漏洞验证结果内容。
6. 调整产品徽章样式，去掉当前偏黄底橘字的视觉问题。

## 为什么

当前详情区有三块信息重复：左侧“识别产品”“开放端口”和右侧“端口-产品-协议”都在展示相近内容，信息密度高但不利于快速浏览。用户更关心的是：当前端口是什么、识别了哪些产品、关联了哪些漏洞、漏洞验证结果是什么。

## 怎么做

1. 后端结果 JSON 改为直接返回可渲染的 `port_overview` 和 `related_vulns`。
2. `port_overview` 以端口行为单位，包含协议、端口、URL、产品列表、是否为当前行。
3. `related_vulns` 只返回与当前结果 target 对应的漏洞插件、CVE 和漏洞结果记录 id，不在首屏直接携带 result 大文本。
4. 前端详情区改成三列：基础信息 / 端口概览 / 受影响漏洞。
5. 新增结果弹窗与交互按钮；点击“查看结果”后再请求漏洞 result 文本，不改现有 HTML 源码查看能力。
6. 为 `auto_scan_exp_result(task_id, target)` 和 `auto_scan_indentify_result(ip, port)` 补充索引，降低资产详情和漏洞结果查询成本。

## 风险

- 漏洞结果和识别结果目前底层仅靠 `task_id + target` 软关联，要注意全局资产页与 standalone 页都返回一致数据。
- 端口概览需要同时兼顾“当前端口”和“其他端口”排序，避免把当前端口埋在列表中。
- 这次是大块模板改造，容易影响现有展开详情布局，需要回归检查分页、展开、HTML 查看按钮。
