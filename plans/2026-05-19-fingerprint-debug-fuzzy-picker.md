# 指纹调试页新增模糊选择器

- 状态：已完成
- 创建时间：2026-05-19

## 做什么

给指纹调试页新增一个可模糊搜索的指纹选择项；用户选中已有指纹后，自动把该指纹值覆盖到当前指纹输入框。

## 为什么

现在指纹调试只能手动输入，容易把已有规则重复敲一遍，也容易输错。增加选择器后，调试现有规则更快，且能减少输入错误。

## 怎么做

1. 在 `fingerprint_debug` 页面加一个可搜索的指纹下拉选择器。
2. 页面加载时一次性带出已有指纹列表，前端本地模糊筛选。
3. 选中某条指纹后，把对应 `condition` 值写回当前指纹输入框。
4. 保持原有“手动输入 + 匹配”流程不变。
5. 补最小回归验证，确认旧的手动输入仍可用，且选择器能覆盖输入框。

## 风险

- 指纹列表量大时，页面初次渲染会变重；先走前端本地模糊搜索，避免新增接口和额外联动复杂度。
- 这是纯 UI 行为，必须避免改动匹配规则本身。
- 需要同步 backlog / changelog / 项目控制台，避免状态落后。

- [x] 选中某条指纹后，`#regex` 输入框被覆盖成该指纹值。
- [x] 手动输入指纹后，原有匹配按钮流程仍可用。
