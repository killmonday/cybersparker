# 2026-05-20 BL-FINGER-004 指纹调试页全库匹配结果列表

## 做什么
- 给指纹调试页增加“是否同时匹配所有库中指纹”勾选框。
- 勾选后，本次识别会把指纹库全部规则都参与匹配。
- 新增一个结果区，展示本次所有命中的指纹名字和对应条件。
- 当前调试的指纹如果命中，在结果区里显示为“当前调试指纹”。
- 补充后端/前端回归测试，确认原有单条调试流程不受影响。

## 为什么
- 现在只能盯着单条规则看，调试一个目标时没法顺手确认“这批库里到底还有哪些规则也命中”。
- 需要把当前调试规则和全库命中结果放在同一页里看，减少反复切换和重复试错。

## 怎么做
1. 在 `app_cybersparker/views/expload/fingerPrint_debug.py` 增加一个开关参数，控制本次只测当前规则还是把全库指纹都纳入识别。
2. 返回给前端的 JSON 里补上命中列表，包含指纹名字、条件和命中片段；当前调试规则命中时，名字固定显示为“当前调试指纹”。
3. 在 `app_cybersparker/templates/project/expload/fingerprint_debug.html` 增加勾选框和命中列表展示区。
4. 在 `app_cybersparker/tests.py` 补最小回归测试：页面元素、单条流程、全库命中列表和“当前调试指纹”标签。
5. 同步 `CHANGELOG.md`、`docs/backlog/03-指纹与自动识别.md` 和 `docs/项目控制台.md`。

## 风险
- 全库匹配会遍历指纹库，数据量大时会比单条调试慢，但这是用户主动勾选后的行为，且只发生在调试页。
- 需要保证“当前调试指纹”这个标签只改展示，不改规则判断本身。

## 验证
- `python manage.py test app_cybersparker.tests.FingerprintDebugPageTests --keepdb --noinput -v 2`
- `python manage.py check`

## 当前状态
- 已完成实现与定向验证，等待文档与审查收尾。

## 结果
- 指纹调试页新增“是否同时匹配所有库中指纹”勾选框。
- 勾选后，后端会在当前调试规则之外，再遍历指纹库并返回所有命中的指纹名字、条件和命中片段。
- 当前调试规则命中时，结果列表里固定显示为“当前调试指纹”。
- 同步修正了旧 `handle()` 在 `1&&2&&(3||4)` 这类混合条件下参数顺序传反的问题，避免全库匹配列表在这类规则下误判。
- 新增 3 条回归测试，覆盖未勾选仅测当前规则、勾选后返回全库命中、当前规则未命中但库内其它规则命中。
