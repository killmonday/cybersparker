# 自动扫描任务批量删除失效修复

- 状态：已完成
- 创建时间：2026-05-20

## 做什么

修复自动扫描任务列表页“批量删除”按钮无效的问题。

## 为什么

前端批量操作请求发到了错误接口，后端自动扫描删除接口也只实现了单条 `uid` 删除，没有接住列表页提交的 `contents[]` 批量删除数据，所以按钮点击后无法真正删除选中的任务。

## 怎么做

1. 把自动扫描列表页批量删除请求改到正确的 `/Identify_task/delete` 接口。
2. 在 `auto_scan_task.delete()` 中补充 POST 批量删除分支，兼容 `contents[]` / `contents` 两种数组参数。
3. 避开同文件 `list()` 视图函数对 Python 内置 `list()` 的遮蔽，防止批量删除分支在取 queryset 时被误调到列表页视图。
4. 复用现有单条删除的目标文件清理逻辑，保持批量删除时也会清理目标文件。
5. 增加定向测试覆盖自动扫描任务批量删除。
6. 自动扫描 `delete` 接口补齐 `@csrf_exempt`，与同目录现有任务管理接口保持一致，避免前端 POST 被 CSRF 中间件拦截。

## 风险

- 批量删除会连同目标文件一起清理，必须保持与单条删除一致，不能误删非当前任务文件。
- 只允许最小修复，不改自动扫描其他批量操作语义。

## 验证

- [x] 自动扫描任务批量删除接口返回成功。
- [x] 被选中的任务记录会被删除。
- [x] 对应目标文件清理函数会被调用。
- [x] `python manage.py check` 通过。

## 结果

- 前端批量删除请求已改到 `/Identify_task/delete`，不再打到不存在的 `/Identify_task_result/operate`。
- 后端 `auto_scan_task.delete()` 已补齐 POST 批量删除分支，并兼容 `contents[]` / `contents` 两种数组参数。
- 已修复同文件 `list()` 视图函数遮蔽 Python 内置 `list()` 导致批量删除分支异常的问题。
- 已新增自动扫描批量删除定向测试，覆盖“任务记录删除 + 目标文件清理调用”这两个关键行为。

## 风险结论

- 本次改动只影响自动扫描任务列表页的批量删除链路，不影响单条删除、任务执行、结果页或其他模块。
- 批量删除仍复用原有单条删除的目标文件清理逻辑，行为保持一致。
