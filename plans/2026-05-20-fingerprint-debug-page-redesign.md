# 指纹调试页视觉重做

- 状态：已完成
- 创建时间：2026-05-20

## 做什么

重做指纹调试页的视觉样式和信息排布，让页面更像调试工作台，而不是普通表单页。

## 为什么

当前页面虽然能用，但层次弱、信息块拥挤、结果区不够直观，整体观感比较旧。这个页面本质上是“输入规则 + 观察响应 + 判断是否命中”的实验台，应该更强调输入区、结果区和状态反馈。

## 怎么做

1. 采用深色实验台风格，强化页面氛围和层次。
2. 重排页面结构：顶部说明区、左侧输入控制区、右侧结果观察区。
3. 优化输入框、下拉框、按钮、响应头/响应体/匹配结果三块结果卡片的样式。
4. 保留现有 URL、已有指纹筛选回填、HTTP 代理、匹配按钮、AJAX 返回处理逻辑。
5. 补最小回归验证，确认页面仍能正常渲染，原有后端接口不受影响。

## 风险

- 这是模板级改动，容易误伤现有 JS 选择器和 DOM id，必须保留关键 id。
- 如果把结构改得太大，可能影响全局后台布局，所以只改当前模板内部。
- 当前没有浏览器实测条件，视觉效果只能做到代码级确认。

## 结果

- 页面改成深色实验台风格，增加了顶部说明区、左侧输入控制区、右侧响应观察区。
- 增加了更清楚的状态提示：待发起、匹配中、命中成功、未命中、规则错误、请求失败。
- 没改后端匹配接口和原有关键 DOM id，已有模糊选择回填能力仍保留。

## 风险复盘

- 当前只做了代码级验证，没有浏览器人工验收，所以视觉细节和实际交互手感还需要浏览器里再看一遍。
