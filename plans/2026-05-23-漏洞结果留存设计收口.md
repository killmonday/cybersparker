# 做什么
更新目录扫描设计文档里关于漏洞结果表的约定：
- 同一资产、同一漏洞的每次成功验证都新增一条记录
- 不再用 upsert 覆盖历史漏洞结果
- 检索页面和详情展示默认取最新结果
- 明确历史结果保留的原因和查询影响

# 为什么
漏洞验证结果会随时间变化，历史结果本身有价值；如果覆盖掉，后续就看不到“当时确实存在过”的证据。

# 怎么做
1. 先把漏洞结果表的写入语义改成追加。
2. 再补一段查询约定，说明列表和详情页默认看最新一条。
3. 最后在变更日志里补一条说明。

# 风险
追加后，同一资产同一漏洞会有多条记录，所有依赖”唯一一条”的查询都要明确选最新记录，否则页面会乱。
