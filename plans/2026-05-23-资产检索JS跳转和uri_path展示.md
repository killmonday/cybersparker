# 做什么
更新 `docs/modules/00-资产检索与目录扫描设计.md`：
- 两个资产检索页面都展示 `uri_path`
- `uri_path` 继续表示自动扫描中最终落地 URL 的 URI path
- 增加 JS 跳转识别
- 多跳场景只对首个响应和最终响应做产品识别

# 为什么
现在的设计只描述了 HTTP 跳转后的结果，没有把页面里常见的 JS 跳转和多跳处理收进去；如果不补，会出现展示和扫描口径不一致。

# 怎么做
1. 在根资产字段说明里补 `uri_path` 的展示口径。
2. 在扫描流程里补 JS 跳转识别规则和多跳处理规则。
3. 在变更日志里补一条记录。
4. 做一次全文检视，确认没有旧说法残留。

# 风险
如果只补一半，可能会把“展示路径”和“扫描路径”混成两种说法，后面实现时容易写歪。
