# BL-AUTO-014 资产检索页面性能优化

- 状态：已完成
- 关联 backlog：`docs/backlog/03-指纹与自动识别.md` → BL-AUTO-014

## 改动总览

### 第一轮：响应体积 + 并行加载

1. JSON 响应去 favicon Base64：只保留 `favicon_md5`
2. 主搜索与聚类统计并行加载

### 第二轮：N+1 与重复查询消除

3. port_overview / related_vulns 从 26 次单查 → 2 次批量查询
4. 去重 `data.count()` 和 `build_facet_result` 调用

### 第三轮：游标分页 + 计数优化

5. Pagination 类重写：`WHERE id < cursor LIMIT n+1` 替代 OFFSET+LIMIT+COUNT
6. 首页 `COUNT(*) OVER()` 一次 SQL 拿结果+精确总数
7. `pg_class.reltuples` 估算值作为 fallback
8. 前端页码导航改为"上一页/下一页"

### 第四轮：懒加载

9. 端口概览：首页只展示当前资产行 + "查看同 IP 其他资产"按钮
10. 无漏洞时隐藏"受影响漏洞"列

### 第五轮：favicon 文件化

11. 采集时写入 `app_cybersparker/static/favicons/<md5>.<ext>`，DB 只存路径
12. post_delete 信号级联清理 favicon 文件
13. 迁移 0034 批量导出已有 base64 为文件

### 数据库索引

14. 迁移 0033：`UPPER(html) gin_trgm_ops` 表达式索引

### 其他

15. `auto_scan_exp_Result.py` CSV 下载加 `select_related('EXP_id')`
16. `all_Indentify_result.py` `get_product()` 从无条件调用改为首次 IP 守卫

## 性能效果

| 指标 | 改前 | 改后 |
|------|------|------|
| 英文搜索总耗时 | ~1000ms | ~280ms |
| JSON 响应体积 | >2MB | ~10KB |
| 主请求 DB 查询数 | ~31 次 | ~4 次 |

## 遗留

- pg_trgm 对中文无效，中文搜索仍走 Seq Scan (~1060ms)
- favicon facet 查询仍走过滤后数据的聚合（~500ms），待改为异步加载
