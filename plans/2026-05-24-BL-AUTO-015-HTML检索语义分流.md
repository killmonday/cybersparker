# BL-AUTO-015 HTML检索语义分流

## 做什么
- 把 `body/html` 检索拆成两条语义：`=` / `:` 走快捷检索（tsvector），`:=` 走子串检索。
- `:=` 子串检索按内容路由：纯英文优先 trigram、纯中文优先 bigm、中英混合交给 PostgreSQL planner/cost 选索引。
- `:=` 模式下 favicon facet 改为异步请求，不阻塞首屏主结果。

## 为什么
- 保留 token 级快捷搜索的高性能。
- 让 HTML 原文精确片段搜索继续可用。
- 深度检索场景下避免 favicon facet 同步聚合拖慢首屏。

## 怎么做
- 同步修改 `auto_scan_result.py` 与 `all_Indentify_result.py` 的解析逻辑与查询分支。
- 调整 standalone / 全局结果页前端请求顺序，使 `:=` 模式下 favicon facet 独立请求。
- 补齐相关测试与文档说明。

## 风险
- 中英混合子串的索引选择依赖 PostgreSQL planner，结果可能随统计信息变化。
- 短子串仍可能产生较多候选行，深度检索在极短 pattern 下不一定稳定低于 500ms。
- `:=` 模式异步化后，前端回退逻辑需要保持可用。

## 验收
- `=` / `:` 的 `body/html` 检索走快捷检索，不再走子串查询。
- `:=` 的 `body/html` 检索走子串查询，纯英文 / 纯中文 / 混合串能按预期命中。
- `:=` 模式下 favicon facet 与主搜索分离，请求不再串行阻塞首屏。
- `python manage.py check` 0 issues。
- 相关回归测试通过。
