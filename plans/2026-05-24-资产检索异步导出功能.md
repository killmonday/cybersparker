# 资产检索页导出功能

## 背景

两个资产检索页面（standalone 任务结果页、全局资产检索页）的"字段"按钮目前形同虚设——弹窗勾选的字段不影响任何渲染。用户需要将其改造为真正的异步导出功能，并新增导出任务管理子页面。

## 目标

把"字段"按钮替换为"导出"按钮，用户选择导出字段和条数上限后提交，后端异步生成 CSV，用户可在新的导出任务管理页查看/下载历史导出。

## 方案设计

### 1. ExportTask 模型

```python
class ExportTask(models.Model):
    task_type = models.CharField(max_length=16)       # 'global' / 'task'
    task_id = models.IntegerField(null=True, blank=True)  # auto_scan_tasks id
    task_name = models.CharField(max_length=256, blank=True)  # 任务名（展示用）
    search_string = models.TextField(blank=True)       # 检索条件
    fields = ArrayField(models.CharField(max_length=32))  # 勾选的导出字段
    include_vuln_result = models.BooleanField(default=False)  # 是否导出漏洞验证结果
    export_limit = models.IntegerField(null=True, blank=True)  # 导出条数上限
    status = models.CharField(max_length=16, default='processing')  # processing/completed/failed
    csv_file = models.CharField(max_length=512, blank=True)  # CSV 文件路径
    total_rows = models.IntegerField(null=True, blank=True)  # 导出行数
    error_message = models.TextField(blank=True)
    creatime = models.DateTimeField(default=timezone.now)
```

### 2. 可选导出字段（完整清单）

在现有 display_fields 基础上补全：

```
title, product, ipc, country, province, city, isp, port, protocol,
status_code, uri_path, host, ip, favicon_md5, cert_org, cert_common_name,
cert_serial, vuln(漏洞名称), cve(CVE编号)
```

vuln_result（漏洞验证结果）不显示在复选框列表中，而是当用户勾选 vuln 时，额外在 CSV 中追加一列"漏洞验证结果"。

### 3. 导出弹窗 UI

替换原来的 settingsModal：
- 标题改为"导出 CSV"
- 所有字段复选框（2 列 grid 布局）
- 新增"导出条数"输入框：默认 1000，下拉可选 100/500/1000/5000/10000/全部，也支持手动输入
- 后端校验：最大值 50000，最小值 1，"全部"不限制
- 当勾选 vuln 时，底部提示"将同时导出漏洞验证结果"
- 确认按钮文字"开始导出"

### 4. CSV 生成逻辑（Celery 任务）

```
1. 根据 task_type 确定 queryset：
   - task: filter(task_id=...) + 解析 search_string
   - global: 解析 search_string 或 all()
2. 如果有 export_limit，[:export_limit]
3. 判断是否需要 JOIN vuln 表：
   - 如果 fields 包含 vuln/cve/vuln_result → 批量 JOIN auto_scan_exp_result + EXP
   - 其他字段直接从 auto_scan_indentify_result 取值
4. 每行 = 一个资产-漏洞组合（无漏洞字段时一行一资产）
5. 写入 CSV 文件到 static/exports/<task_id>_<timestamp>.csv
6. 更新 ExportTask 状态为 completed 或 failed
```

关键实现细节：
- CSV 列顺序 = fields 勾选顺序，vuln_result 追加在 vuln 列之后
- vuln 列输出 `EXP.title`，cve 列输出 `EXP.CVE`
- 同一资产有多个漏洞时，资产信息列重复，漏洞列不同
- 如果资产没有任何关联漏洞但勾选了 vuln 字段 → 漏洞列留空

### 5. 导出任务列表页

模板：`export_task_list.html`（继承 `project/index.html`）

每行显示：
- 任务类型（全局/任务名）
- 检索条件（截断）
- 导出字段
- 状态（处理中/已完成/失败）
- 总行数
- 创建时间
- 操作按钮：下载（completed 时可用）、重试（failed 时可用）

### 6. URL 路由

| 路由 | 名称 | 说明 |
|------|------|------|
| `POST /asset/export` | `asset_export` | 提交导出任务（支持 global + task） |
| `GET /export/tasks/` | `export_task_list` | 导出任务列表页 |
| `GET /export/tasks/<id>/download` | `export_task_download` | 下载 CSV 文件 |

### 7. 前端交互流程

1. 用户点击"导出"→ 弹出模态窗口
2. 勾选字段 + 输入导出条数 → 点击"开始导出"
3. JS 发送 POST 到 `/asset/export`，携带 `{fields, export_limit, search_string, task_type, task_id}`
4. 后端创建 ExportTask + dispatch Celery → 返回 `{status: 'ok', task_id: N}`
5. 前端显示 toast "导出任务已提交，请在导出任务页查看"
6. 用户在导出任务页看到任务列表，完成后点击下载

### 8. 后端校验

- `export_limit`：min=1, max=50000，不传或传 "all" 表示不限制
- `fields`：必须是非空数组，每个值必须在白名单内
- `task_type`：必须是 'global' 或 'task'
- `task_id`：task_type='task' 时必须提供且存在

## 不做

- 不改造旧的 inline 结果页（`auto_scan_identify_result.html`）
- 不删除 display_fields 全局变量（独立清理）
- 不做 Excel 格式导出（仅 CSV）
- 不提供导出任务删除/清理功能（后续迭代）
