# 端口概览首屏不加载修复

- 日期：2026-05-25
- 关联 Backlog：无（Mode D-lite 例外，小型行为修复）
- 状态：已完成

## 问题

两个资产检索页面首屏加载时，端口概览区域均显示"暂无端口概览"：

| 页面 | URL | 视图 |
|------|-----|------|
| 单任务独立结果页 | `/Identify_task/<id>/result/standalone` | `Task_result` |
| 全局资产检索页 | `/asset/search` | `global_asset_search` |

点击检索按钮（AJAX JSON 路径）后正常，刷新页面又消失。

## 根因

两个视图的 HTML 渲染路径都使用 `_result_items.html` 模板，模板检查 `{% if item.port_overview %}`。但视图中只对 items 设置了 `related_vulns` 和 `country_code`，从未填充 `port_overview` 字段。

JSON/AJAX 路径不受影响，因为 JS 端 `renderPortOverview()` 直接使用 `item.protocol`/`item.port` 等字段渲染。

## 修复

### 1. 新增 `_attach_port_overview(items)` helper（`auto_scan_result.py`）

为每个 item 填充当前资产端口信息 + "查看同 IP 其他资产"按钮标记，与 JS `renderPortOverview()` 行为一致。

### 2. `Task_result` 和 `global_asset_search` 的 HTML 渲染路径各加一行调用

```python
_attach_port_overview(items)
```

### 3. `_result_items.html` 两处调整

- 按钮文案从"展开更多（共 X 项）"改为"查看同 IP 其他资产"
- 无漏洞时整列隐藏（`{% if item.related_vulns %}` 包裹 vuln-col），与 JS `renderResults` 一致

## 涉及页面

| 页面 | URL | 说明 |
|------|-----|------|
| 单任务独立结果页 | `/Identify_task/<id>/result/standalone` | 查看单个识别任务的结果 |
| 全局资产检索页 | `/asset/search` | 跨任务检索所有资产 |

## 风险

- 极低：仅给每个 item 加了 1 个含 1 行 dict 的 list，不触发额外 DB 查询。
