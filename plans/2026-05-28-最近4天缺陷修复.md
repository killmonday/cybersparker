# 最近4天缺陷修复（2026-05-28）

- 状态：部分完成
- 范围：基于 `plans/2026-05-28-最近4天变更审查报告.md` 修复高优先级问题，并同步必要文档。

## 做什么

1. 修复资产导出仍读取旧 `task_id` 字段的问题。
2. 修复检索语句输入源任务在重跑/重启时未重置 `last_id` 的问题。
3. 修复自动扫描任务详情接口未返回 `search_query` 等字段的问题。
4. 复核并保留 `batch_task_executor.py` 当前未提交修复，只在必要时补齐同一链路问题。
5. 同步受影响 backlog / 模块文档 / CHANGELOG。

## 为什么

这几处不是样式问题，而是会直接导致导出失败、任务空跑、编辑表单前后端不一致的问题，优先级高于界面细节。

## 怎么做

### 1. 导出链路
- 改 `app_cybersparker/tasks.py:run_export_task`
- 不再读取 `item.task_id`
- 通过 `AssetTaskRelation` 为每条资产取所属任务集合，再按 `(task_id, target)` 关联漏洞结果
- 补测试：导出时包含 vuln/cve + 同资产多任务场景

### 2. 检索语句任务重跑/重启
- 改 `auto_scan_task.py:Task_operate`
- 改 `batch_exp_task.py:operate`
- 对 `input_type=6` 的 start/restart/rerun 场景强制 `last_id=0`
- 保留 resume 语义：resume 继续跑，不清游标
- 补测试：auto_scan / batch 两条线至少覆盖“跑完后重跑仍会重新消费目标”

### 3. auto_scan 详情接口
- 改 `auto_scan_task.py:detail`
- 补回 `search_query/parsed_query/frozen_max_id/last_id`
- 补测试：detail 返回字段完整

### 4. 文档同步
- 更新 backlog 与模块文档中已确认的行为描述
- 记录“筛选支持两种 EXP”的统一口径
- 更新 CHANGELOG

## 风险

1. 导出链路会碰到多任务共享同一资产，需要避免把别的任务漏洞混进当前导出。
2. `resume` 和 `rerun/restart` 语义不能混，避免把续跑也重置成从头开始。
3. 文档同步只改已确认偏差，不顺手扩写无关内容。
