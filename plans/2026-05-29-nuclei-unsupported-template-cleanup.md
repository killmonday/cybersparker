# 2026-05-29 Nuclei 不支持协议模板清理与导入过滤

## 做什么
- 清理数据库中所有带当前引擎不支持协议块的 nuclei 模板。
- 更新 `import_nuclei_templates`，以后导入时直接跳过这类模板。
- 补充相关文档，记录协议类型解释、本次决策和实际处理结果。

## 为什么
- 当前引擎只支持 HTTP 与 TCP/network 两类 nuclei 执行路径。
- 现网库里已经混入 `code/javascript/headless/file/dns/ssl/websocket/whois` 等协议模板，会造成模板列表脏数据、任务误选和理解混乱。
- 用户已明确要求：所有不支持类型模板全部删除，并阻止未来再次导入。

## 怎么做
1. 在运行时模块中抽出统一的“不支持协议识别”函数。
2. 在 `import_nuclei_templates.py` 里解析 YAML 后先判定协议；命中不支持协议则跳过，不创建 EXP。
3. 新增清理命令：扫描全部 `plugin_language=2` 模板，删除命中不支持协议的 EXP、关联文件、关联关系，并清理批量任务里残留的 EXP id 字符串引用。
4. 补定向测试，覆盖：协议识别、导入跳过、清理命令删除与任务引用清理。
5. 实际执行清理命令并记录删除数量。
6. 同步 backlog、模块文档、总览、恢复指南、CHANGELOG、项目控制台。

## 风险
- 删除是不可逆的本地数据操作，但范围明确：仅 `plugin_language=2` 且 YAML 顶层包含不支持协议块的模板。
- `batch_EXPTask.EXP` 不是外键，需额外清理逗号串里的待删 id，避免留下悬空引用。
- 旧结果表若外键指向待删模板，会随 EXP 级联删除；执行前需确认实际数量。

## 验证
- 定向测试通过。
- `python manage.py check` 通过。
- 清理命令 dry-run/实跑输出与数据库复核一致。

## 结果
- 进行中。

## 后续
- 若后续要支持 `dns/ssl/headless/...`，应先补对应执行器，再决定是否重新导入被跳过模板。
