# BL-FE-001 鉴权、CSRF 与 API 契约冻结

- 状态：已完成
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md#bl-fe-001-鉴权csrf-与-api-契约冻结`
- 分支：`feat/react-frontend-separation`

## 做什么

梳理现有登录态、鉴权中间件、CSRF 使用方式和首批低风险列表页接口现状，产出 React 首批接入可执行的 `/api/v1` 契约口径，并在必要时做最小代码/文档收口。

## 为什么

如果不先冻结这些规则，React 页面即使先做出来，也会因为未登录返回不统一、CSRF 口径混乱、旧接口返回形状不稳定而频繁返工。

## 怎么做

1. 梳理登录态建立方式、鉴权拦截方式、深链刷新风险。
2. 盘点首批低风险列表页的现有 HTML/JSON 接口、分页和筛选参数。
3. 盘点 CSRF 中间件是否启用、哪些历史 POST 接口已豁免。
4. 形成 `/api/v1` 约束：未登录返回、分页参数、错误码、CSRF 规则、禁止直连的旧接口类型。
5. 如有必要，补最小文档或最小后端壳子改动，为 BL-FE-002 铺路。

## 风险

- 现有登录态可能依赖自定义 session 字段，不是标准 Django auth。
- 历史页面可能大量依赖 `csrf_exempt`，不能直接照搬到 React。
- 同一功能可能同时存在 HTML 页面、AJAX 接口和表单提交，口径不一定一致。

## 验证

- `python manage.py check`
- 文档中明确未登录返回、分页、错误码、CSRF 口径
- 至少覆盖 1 个受保护 GET 和 1 个受保护 POST 的接入规则

## 当前进度

- 2026-05-30：开始执行，已并行启动登录鉴权、接口现状、CSRF 现状三路梳理。
- 2026-05-30：登录鉴权梳理完成。当前登录由 `app_cybersparker/views/login.py` 里的 `/login` 固定账号密码校验后写 `request.session["info"]` 建立；`logout()` 存在但未在 `cybersparker/urls.py` 注册。`AuthMiddleware` 仅显式放行 `/login`，`BasicAuthMiddleware` 会对已注册 URL 再做一层 BasicAuth，导致 React 内容区如果挂在已注册页面上，刷新/直达会同时受 session 与 BasicAuth 影响；History 深链到未注册前端路由则会直接 404。首版 `/api/v1` 应统一未登录返回 `401 JSON`，不能混用 302/HTML。
- 2026-05-30：CSRF 梳理完成。全局已启用 `CsrfViewMiddleware`，但历史上已有大量 `@csrf_exempt` 写接口，首批低风险页面相关的 add/edit/delete 也有不少在豁免名单内。结论：React 首批应优先只接 GET 列表/详情；写操作不要直连历史 `@csrf_exempt` 接口，优先继续保留 Django 表单，或新建同源 `/api/v1` 写接口并统一带 CSRF cookie + `X-CSRFToken`。调试执行、任务启停、结果删除下载、历史文件清理类接口列入 React 首批禁止直连清单。

## 首版结论

1. 现有后台登录不是标准 Django auth 登录流，而是 `/login` 写 `request.session["info"]`；同时所有已注册 URL 还会再过一层 BasicAuth。
2. React 第一阶段不能直接放开 History 深链，必须让每个页面先有一个真实 Django 壳页路由。
3. `/api/v1/**` 未登录返回必须统一成 `401 JSON`，不能混用 HTML 跳转。
4. 全局已启用 `CsrfViewMiddleware`，但历史 `@csrf_exempt` 写接口很多，React 首批禁止直连这些旧写接口。
5. 首批低风险页面优先只做 GET 列表/详情；真正写操作优先保留旧页入口，或后续单独补 `/api/v1` 写接口。
6. 首批建议接口：`/api/v1/auth/session`、`/api/v1/plugins`、`/api/v1/plugins/{id}`、`/api/v1/dict-groups`、`/api/v1/dicts`、`/api/v1/fingerprints`、`/api/v1/proxies`、`/api/v1/cyberspace-engines`。
7. UI 策略确定：不套整套第三方后台模板，只借组件能力，不借整站骨架；现有左菜单 + 顶部栏 + 内容区结构保持不变。

## 已完成实现

- 新增 `GET /api/v1/auth/session` 会话探针接口：未登录返回 `401 JSON`，已登录返回 `authenticated + user`。
- `AuthMiddleware` 对 `/api/v1/**` 统一改为未登录返回 `401 JSON`，非 API 页面仍沿用跳 `/login`。
- 补注册 `/logout` 路由。
- `BasicAuthMiddleware` 对 `/login`、`/logout` 放行；若 session 已存在则不再重复挑战 BasicAuth，避免 React 壳页页内请求被反拦截。
- 新增 `AuthContractTests` 5 条定向测试，覆盖会话探针、API 401 口径、BasicAuth 放行规则。
