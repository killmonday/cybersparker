# BL-FE-002 React 基线接入、Django 壳页挂载与静态资源出口收口

- 状态：已完成
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md#bl-fe-002-react-基线接入django-壳页挂载与静态资源出口收口`
- 分支：`feat/react-frontend-separation`

## 做什么

新增一个最小 React + Vite + TypeScript 前端工程，输出到 Django 可直接加载的静态目录；同时新增一个 Django 壳页路由和模板，把 React 先挂进内容区，验证登录后可进入 React 页面。

## 为什么

BL-FE-001 已把鉴权、CSRF 和 `/api/v1` 口径收口了；现在需要一个最小可运行的前端基线，证明 React 工程、Django 模板壳页、静态产物出口三件事能够连起来。

## 怎么做

1. 新建 `frontend/`，落 React + Vite + TypeScript 最小工程。
2. Vite 构建产物输出到 `app_cybersparker/static/react-shell/`，作为唯一静态出口。
3. 新增 Django 壳页模板，复用现有 `project/index.html` 母版，只在内容区挂一个 React 根节点。
4. 新增 Django 视图和路由，登录后可通过真实 Django URL 进入壳页。
5. React 页面最小验证调用 `GET /api/v1/auth/session`，显示会话状态。
6. 跑前端构建、Django 检查和壳页入口相关验证。

## 风险

- Node/Vite 产物命名如果不稳定，Django 模板引用会容易失效。
- 壳页入口若直接改母版导航，容易影响全部旧页面；本次应避免。
- 如果直接接管 History 路由，刷新会撞上 Django 404；本次不做。
