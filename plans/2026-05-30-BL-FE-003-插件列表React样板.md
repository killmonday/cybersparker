# BL-FE-003 首批低风险列表页迁移（插件列表样板）

- 状态：已完成
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md#bl-fe-003-首批低风险列表页迁移`
- 分支：`feat/react-frontend-separation`

## 做什么

先不同时迁多个页面，先把插件列表页做成首个 React 样板页：补一个只读列表 API、做一个 React 列表页面、给它一个 Django 壳页入口，验证 BL-FE-001 + BL-FE-002 的基线已经能支持真正业务页面迁移。

## 为什么

插件列表页交互复杂度适中，既有分页筛选，也有详情跳转和后续编辑入口，适合做第一块样板。只要这条链路跑通，后面的字典列表、指纹列表、代理设置列表都可以按同样模式推进。

## 怎么做

1. 梳理现有 `expload/list` 的字段、搜索参数、分页参数。
2. 新增只读 API：返回插件列表、分页信息、筛选参数回显。
3. 在 `frontend/` 中新增插件列表页面组件，先只读展示。
4. 新增 Django 壳页入口，比如 `/react-shell/plugins`。
5. React 页面调用新 API，展示列表、筛选、分页和跳旧页编辑/详情的链接。
6. 跑构建、Django 检查和定向测试。

## 风险

- 不能直接复用旧 HTML 列表视图，需要补稳定 JSON。
- 不能过早把新增/编辑/删除一起搬进来，否则会把 CSRF 和旧接口问题重新带回来。
- 插件列表模板里现有字段较多，第一版要克制，只做最必要字段。

## 验证

- `npm --prefix frontend run build`
- `python manage.py check`
- 必要时补插件列表 API/壳页定向测试
