# BL-FE-004 结果页接口抽离但不换 UI

- 状态：已完成
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md#bl-fe-004-结果页接口抽离但不换-ui`
- 分支：`feat/react-frontend-separation`

## 做什么

先不改现有结果页和资产检索页的 Django 模板 UI，只抽离它们背后的 JSON / 分页 / facet / 查询参数口径，为后续 React 化打基础。

## 为什么

结果页是当前项目里最复杂的一块：有 standalone 结果页、全局资产检索、facet、分页、详情展开、结果下载等逻辑。如果一边改 UI 一边梳理接口，很容易反复返工。

## 怎么做

1. 盘点结果页相关视图、路由、搜索参数、分页参数和 JSON 返回结构。
2. 划分“旧模板继续用”与“先抽成稳定 JSON”的边界。
3. 统一字段命名、错误返回、facet 参数和分页参数。
4. 在不改 UI 的前提下，补必要的稳定 JSON 契约层。
5. 用定向测试或最小验证证明旧页行为未退化。

## 风险

- 结果页字段很多，容易因为收口过度把现有页面打坏。
- standalone 与全局检索页共享逻辑，抽接口时必须保持两个入口口径一致。
- 不能改动现有搜索语义。

## 验证

- `python manage.py check`
- 结果页相关定向测试 / 验证
- 旧模板页面继续可用

## 第一版接口边界清单

### 1. 任务内结果 JSON
- 路由：`GET /Identify_task/<uid>/result?format=json`
- 现状：`Task_result()` 同时承担 HTML 页面和 JSON 返回。
- 当前关键参数：`search_data`、`favicon_offset`、分页参数（复用 `Pagination`）
- 当前 JSON 关键字段：`results`、`has_next`、`has_prev`、`next_cursor`、`prev_cursor`、`page_size`、`estimated_total`、`exact_total`、`favicon_items`、`favicon_has_more`
- 风险：和 standalone HTML 共用一套查询、related_vulns、port_overview 逻辑，不能拆坏。

### 2. 全局资产检索 JSON
- 路由：`GET /asset/search?format=json`
- 现状：`global_asset_search()` 同时承担 HTML 页面和 JSON 返回。
- 当前关键参数：`search_data`、`favicon_offset`、分页参数
- 当前 JSON 关键字段：与任务内结果 JSON 基本同构，但不带 task 作用域。
- 风险：和任务内结果页共享字段口径，后续收口应保持同构，不要一边叫 `results` 一边叫 `items`。

### 3. Facet JSON
- 路由：`GET /Identify_task/<uid>/facet`、`GET /asset/facet`
- 当前关键参数：`field`、`search_data`、`offset`
- 当前返回：`status`、`field` 加 `build_facet_result()` 的分页片段。
- 约束：`field` 白名单必须保持一致；搜索语法解析失败统一返回 `400 JSON`。

### 4. 漏洞结果/详情 JSON
- 路由：`GET /identify_result/<result_id>/vuln-result`
- 现状：返回单条漏洞文本、插件名、CVE。
- 用途：结果卡片详情补全，不应混入列表分页口径。

### 5. 导出/源码查看接口
- 路由：`GET /Identify_task_Result/download`、`GET /identify_result/<id>/html`
- 现状：前者是 CSV 下载，后者是 HTML 源码纯文本查看。
- 口径：这两类先保持非 JSON，不纳入 React 列表 API 契约，但要记录为结果页依赖入口。

## 第一版统一口径

1. 搜索参数统一记为 `search_data`（结果页先不强改成 `q`，避免打坏旧 UI）。
2. facet 分页统一继续用 `offset`。
3. 结果列表 JSON 统一保留 `results`，不强行改名为 `items`。
4. 搜索语法解析失败统一返回 `400/False JSON`，不能一部分吞掉错误、一部分回空列表。
5. HTML 页面继续走旧模板；本阶段只补文档与必要的稳定 JSON 契约，不改 UI。
