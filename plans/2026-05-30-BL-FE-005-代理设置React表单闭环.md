# BL-FE-005 中风险表单页迁移（代理设置 React 闭环）

- 状态：已完成
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md#bl-fe-005-中风险表单页迁移`
- 分支：`feat/react-frontend-separation`

## 做什么

先不同时迁多个表单页，而是先把代理设置做成第一个 React 可写闭环：在现有只读代理列表样板的基础上，补详情/新增/编辑的受保护 `/api/v1` 接口，并在 React 页面中接通表单提交。

## 为什么

代理设置字段最少，链路最短，又能验证 BL-FE-001 里定下的写接口口径（同源 cookie + CSRF）是否真的可用，适合作为中风险表单页的第一块样板。

## 怎么做

1. 梳理现有 `proxy_setting` 的表单字段、详情接口和保存逻辑。
2. 新增受保护 `/api/v1/proxies/{id}` 详情接口。
3. 新增受保护 `/api/v1/proxies/create`、`/api/v1/proxies/{id}/update` 写接口，统一走 CSRF。
4. 在 React 代理列表页中增加“新增 / 编辑”表单能力。
5. 成功提交后刷新列表，不改旧 `proxy_setting` 页行为。
6. 跑前端构建、Django 检查和定向测试。

## 风险

- 不能直接复用旧 `@csrf_exempt` 写接口给 React。
- 代理保存后会触发 `refresh_conf_from_db()`，需要保证新接口仍沿用这一行为。
- 表单提交必须明确失败提示，避免 React 页 silent fail。
