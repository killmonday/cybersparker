# BL-FE-102 全局资产检索页 React 化（仅跨任务检索主链路）

- 状态：已完成
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md#bl-fe-102-全局资产检索页-react-化仅跨任务检索主链路`
- 分支：`feat/react-frontend-separation`

## 做什么

将 `/asset/search` 的主内容区切到 React，直接消费 `/api/v1/assets/search` 与 `/api/v1/assets/facets` 契约，覆盖跨任务搜索、分页、facet、结果列表和详情展开主链路。

## 为什么

BL-FE-101 已经把 TaskResultsPage 的结果卡片、facet、分页、详情展开骨架做完了。全局资产检索页的 API 响应格式和 TaskResultsPage 完全一致（都用 `_build_identify_results_payload`），唯一区别是 scope=global、API 路径不同、没有 task_id。可以直接复用同一套 React 组件模式。

## 怎么做

1. 后端新增 `react_global_asset_search` 视图和 `/react-shell/assets/search` 路由
2. 前端新增 `GlobalAssetSearchPage` 组件，复用 TaskResultsPage 的结果卡片/facet/分页/详情展开骨架
3. 侧边栏"资产检索"入口切到新 React 路径（保留旧 `/asset/search` 为回退）
4. 补测试、构建、类型检查

## 风险

- 最容易把 task scope 和 global scope 串线 — 需要确保 API URL 不带 task_id
- 侧边栏入口改动影响面小但显眼，切错会导致所有页面的导航失效
