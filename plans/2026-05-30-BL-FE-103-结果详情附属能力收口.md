- 分支：`feat/react-frontend-separation`

## 做什么

在 FE-101/FE-102 已做好的 React 结果页基础上，补齐 3 类详情附属能力：
1. HTML 原文查看 — 展开详情后显示"查看 HTML"按钮，新标签页打开
2. 漏洞结果查看 — 关联漏洞旁显示"查看结果"按钮，弹窗展示漏洞详情
3. 端口概览懒加载 — 展开详情后显示当前端口 + "查看同 IP 其他资产"懒加载按钮

## 为什么

FE-101/FE-102 只覆盖了主链路（搜索、facet、分页、结果列表、基础详情展开），但旧 standalone 模板里详情区还有 HTML 查看、漏洞结果弹窗、端口概览懒加载这 3 个入口。不补齐的话 React 页面的结果详情是不完整的。

## 怎么做

1. 两个结果页（TaskResultsPage / GlobalAssetSearchPage）的展开详情区已完全相同 → 各补 3 个能力
2. HTML 原文查看：展开详情中加"查看 HTML"按钮 → `window.open('/identify_result/<id>/html', '_blank')`
3. 漏洞结果查看：每个关联漏洞旁加"查看结果"按钮 → fetch `/identify_result/<vulnId>/vuln-result` → 弹窗展示
4. 端口概览懒加载：展开详情中加端口概览区 → 显示当前端口 → "查看同 IP 其他资产"按钮 → fetch `/asset/port_overview_more`

## 风险

- 端口概览 API 需要 ip/offset/limit/current_target 参数，确认 TaskResultItem 已包含 ip 字段（是的，已包含）
- 漏洞结果弹窗需要处理加载/空/错误三态
- 两个页面都需改，容易漏改一个
