# 阶段八：结果展示线 React 化规划

- 状态：已完成
- 关联模块：`docs/modules/08-前后端分离模块.md`
- 关联 Backlog：`docs/backlog/08-前后端分离改造.md`
- 前置阶段：阶段七（前后端分离方案 A 基线）

## 阶段目标

在不改动 Django 调度、扫描执行、结果写库和搜索语义的前提下，把 standalone 结果页和全局资产检索页的主内容区逐步切到 React，并复用阶段七已经固定好的 `/api/v1` 结果与 facet 契约。

## 纳入范围

1. standalone 结果页 React 主链路
2. 全局资产检索页 React 主链路
3. 结果详情附属能力收口（HTML 原文查看、漏洞结果查看、端口概览懒加载）

## 不纳入范围

1. 不改搜索 DSL / `search_data` 语义
2. 不改任务控制页（启动、暂停、续跑、重跑）
3. 不改插件调试页、指纹调试页
4. 不在本阶段引入新的写接口
5. 不把导出链路并入本批次

## 执行顺序

1. BL-FE-101 standalone 结果页 React 化（仅任务内结果主链路）
2. BL-FE-102 全局资产检索页 React 化（仅跨任务检索主链路）
3. BL-FE-103 结果详情附属能力收口（HTML / 漏洞结果 / 端口概览）

## 阶段验收

- standalone 结果页与全局资产检索页都能由 React 主内容区稳定展示结果列表
- 同一查询条件下，新旧页结果总数、首屏列表、facet 统计口径保持一致
- task 范围和 global 范围不串线
- HTML 原文查看、漏洞结果查看、端口概览懒加载在新旧页行为一致
- `python manage.py check` 0 issues

## 风险

- 结果页共享模板拆分时，容易把 task scope 与 global scope 混掉
- facet、分页、详情展开状态较多，React 迁移时容易出现状态丢失
- 附属能力虽然不改后端语义，但旧模板里挂着很多隐式入口，迁移时容易漏接

## 回退策略

1. 每一页仍保留 Django 壳页与旧模板回退路径
2. React 只接现有 `/api/v1` 契约，不改后端搜索语义
3. 若 FE-103 附属能力收口不稳，可单独延后，不阻塞 FE-101 / FE-102 主链路验收

## 阶段收口

- 2026-05-30：BL-FE-101（standalone 结果页）、BL-FE-102（全局资产检索页）、BL-FE-103（结果详情附属能力）全部完成，阶段八达到完成线。standalone 结果页与全局资产检索页的主内容区均已切到 React，HTML 原文查看、漏洞结果查看、端口概览懒加载已在两个结果页的展开详情中补齐。所有验证通过。
