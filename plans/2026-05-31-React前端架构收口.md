# React 前端架构收口 — 巨石拆分 + antd 接入

## 问题

`frontend/src/main.tsx` 3788 行 / 183KB 单体文件，16 个页面全塞在一起。没有组件拆分、没有路由库、没有 UI 组件库、没有代码复用。每改一个页面要在 3788 行里定位，且改动可能影响其他页面。

## 目标

将单体巨石拆分为标准 React 项目结构，接入 antd 作为 UI 组件库，建立可复用模式。

## 核心决策

- **路由**：BrowserRouter（basename=/react-shell），Django 侧加一条 catch-all URL
- **UI 库**：antd 5.x（Table/Form/Modal/Button/Select/Pagination/Tag/Space/Card）
- **API 层**：统一 fetch 封装，自动带 CSRF token 和 credentials
- **类型**：按模块拆分到 `types/` 目录
- **页面**：每页一个文件，按路由懒加载（React.lazy）

## 目录结构

```
frontend/src/
  types/          # TypeScript 类型，按模块分文件
  api/            # API 函数 + fetch 封装
  components/     # 共享组件
  pages/          # 页面组件，一个文件一页
  hooks/          # 共享 hooks
  App.tsx         # 路由定义
  main.tsx        # 入口（10 行）
  styles.css      # 全局样式（精简）
```

## 实施步骤

1. 安装依赖（react-router-dom, antd, @ant-design/icons）
2. 抽取类型文件（types/）
3. 创建 API client 层（api/）
4. 创建共享 hooks（useApiList, useFormModal）
5. 创建 AppLayout 共享组件
6. 创建 App.tsx + 更新 main.tsx 入口
7. 逐页转换（16 页 → 3 批）
8. Django 侧加 catch-all URL
9. 构建 + 类型检查
10. 验证所有 16 页面可访问

## 风险

- antd 样式可能与现有 styles.css 冲突 → 清理 styles.css 中已由 antd 覆盖的样式
- BrowserRouter 需要 Django catch-all → 新增一条 URL 规则，旧 URL 保留
- 页面功能回归 → 保持旧 main.tsx 为备份，逐页对比验证
