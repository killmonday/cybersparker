# 前端 UI 现代化改造 — 浅色系

- 日期：2026-06-02
- 模块：前端（React + antd）
- 状态：已完成

## 问题

当前前端 UI 虽已有完善的 CSS token 体系，但视觉效果偏老派：
- 字体用系统默认，缺乏辨识度
- 配色偏暖灰，略显沉闷
- 阴影层级少（仅 2 级），视觉深度不足
- 无动效细节，页面切换生硬
- antd 组件使用默认主题，未充分定制

## 方案

**设计方向**：「精密工具风」— 专业安全工具的现代浅色系 UI。干净克制、字体有辨识度、微妙深度、流畅过渡。

### 实际改动

1. **styles.css**（设计 tokens + 组件样式）
   - 引入 Plus Jakarta Sans 字体（Google Fonts，400/500/600/700），保留 JetBrains Mono 代码字体
   - 色板：暖灰 → 冷灰（#f8f9fa ~ #0d1117），品牌蓝 #2563eb → #3366ff
   - 语义色升级：success #00a86b、warning #e67e22、danger #e74c3c、info #6c5ce7
   - 阴影：2 级 → 5 级（xs/sm/md/lg/xl），更细腻的层级
   - 暗色主题：从土灰暖色 → GitHub 暗色风格（#0d1117 ~ #f0f6fc）
   - 新增页面入场动画 `pageEnter`（淡入 + 上移 6px）
   - 主按钮 hover 上浮效果 + 彩色阴影
   - body 添加极淡噪点纹理

2. **App.tsx**（antd ConfigProvider theme）
   - fontFamily → Plus Jakarta Sans
   - colorPrimary → #3366ff，语义色全部同步
   - 细化组件覆盖：Table header/border/hover、Menu 选中态/边距、Button 阴影、Input focus ring、Card 圆角
   - Layout token 同步暗色主题变化

## 不改

- 功能逻辑和 API 契约
- 页面组件内部结构
- 暗色模式
- 资产检索等独立页面（仅用 antd token 间接影响）
