# 任务系统代理统一控制

## 目标

统一所有后台任务（requests 和 aiohttp）的代理控制：
- 任务配置了代理 → 所有请求走任务配置的代理
- 任务没配置代理 → 全部直连，不走 OS 环境变量代理
- 同一进程内不同线程的任务代理互不干扰

## 设计：用 thread-local 替代全局 conf.proxies

**现状问题**：`conf.proxies` 是进程级全局变量，存在三个缺陷：
1. 同一进程内两个任务线程并发跑时互相覆盖
2. 需要 save/restore，徒增复杂度
3. web 进程里 `refresh_conf_from_db()` 设的值没人读（web 进程内的 HTTP 调用都传了显式 proxies）

**新方案**：用 `threading.local()` 存储当前线程的任务代理。关键设计——用 sentinel 区分两种状态：

- `None` = 未设置（非任务线程），走兼容路径
- `{}` = 任务配置了"不使用代理"，强制直连
- `{"http": "...", "https": "..."}` = 任务配置了具体代理

```python
# hook_request.py
_task_proxy = threading.local()

def set_task_proxy(proxies_dict):
    """任务执行器入口调用。proxies_dict={} 表示任务配置了不使用代理。"""
    _task_proxy.proxies = proxies_dict

def _get_task_proxy():
    """None = 未设置（非任务线程）；{} = 任务线程内配置了不使用代理。"""
    return getattr(_task_proxy, 'proxies', None)
```

hook 第 80-84 行改为：

```python
task_proxy = _get_task_proxy()
if task_proxy is not None:
    # 任务线程内：任务配置覆盖一切（包括插件显式传的 proxies）
    proxies = dict(task_proxy)
elif proxies is None:
    # 非任务线程 + 没传 proxies：走系统默认（向后兼容）
    proxies = dict(conf.get("proxies", {}))
if not proxies:
    proxies = {"http": None, "https": None}
```

### 预期行为验证

| 场景 | 任务配置 | 插件传 proxies | hook 最终值 | 结果 |
|------|---------|---------------|-----------|------|
| 任务线程 | 代理 A | `None` | 代理 A | 走任务代理 ✓ |
| 任务线程 | 代理 A | `{}` | 代理 A | 忽略插件，走任务代理 ✓ |
| 任务线程 | 代理 A | `{'http':'evil'}` | 代理 A | **忽略插件**，走任务代理 ✓ |
| 任务线程 | 不使用 | `None` | `{"http":None,...}` | 直连 ✓ |
| 任务线程 | 不使用 | `{'http':'evil'}` | `{"http":None,...}` | **忽略插件**，直连 ✓ |
| 非任务线程 | — | 显式值 | 显式值 | 搜索引擎等不受影响 ✓ |
| 非任务线程 | — | `None` | conf 系统默认 | 向后兼容 ✓ |

### 代理 dict 构造统一

所有执行器统一使用 `_build_proxy_from_setting()` 构造代理 dict（返回 `{"http": url, "https": url}` 或 `{}`）。该函数处理 http/socks5 映射，确保格式一致。

| 执行器 | 输入 | 构造方式 |
|--------|------|---------|
| batch_task_executor | ProxySetting object | `_build_proxy_from_setting(proxy_obj)` |
| auto_exp_task (run) | self.proxies (dict, 已在 __init__ 构造) | `dict(self.proxies)` — 等价格式 |
| auto_exp_task (exp_consumer) | 同上 | `dict(self.proxies)` |
| dirscan_worker | task.proxy (ProxySetting object) | `_build_proxy_from_setting(task.proxy)` |

### 6. 指纹调试页 — 修复代理控制

- **文件**：`app_cybersparker/views/expload/fingerPrint_debug.py`
- **现状**：第 485 行 `client_kwargs` 里用户选了代理才设 proxy，**没设 `trust_env=False`**，导致用户没选代理时 httpx 穿透到 OS 环境变量代理
- **改动**：第 485 行加 `"trust_env": False`
  ```python
  # 旧
  client_kwargs = {"verify": False, "timeout": httpx.Timeout(12), "headers": requests_headers()}
  # 新
  client_kwargs = {"verify": False, "timeout": httpx.Timeout(12), "headers": requests_headers(), "trust_env": False}
  ```
- 效果：用户选了代理 → 走代理；没选 → 直连。统一了"不设代理就不走 OS 代理"的规则。

### 不做

- 不删 `conf.proxies` 定义（兼容性）
- 不删 `refresh_conf_from_db()` 中的 proxies 赋值（标记废弃）
- 不引入 save/restore（线程局部天然隔离）
- 不改测绘引擎代理体系（独立配置，不在任务代理控制范围内）

### 风险

- socks5 代理：requests 侧依赖 PySocks，aiohttp 侧依赖 aiohttp-socks。两者均已在 `requirements.txt` 中声明。本次改动不引入新依赖。部署时需确认 `pip install -r requirements.txt` 已执行。
- Celery worker fork 后父进程的 thread-local 不会被子进程继承。各执行器在 `run()` 内自行设置 `set_task_proxy()`，不需要跨进程传递。
