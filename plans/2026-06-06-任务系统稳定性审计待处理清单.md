## 批量任务

### BL-AUDIT-001 批量 consumer_exp 暂停时不加速排空

- 文件：`batch_task_executor.py:709-762`
- 现象：消费者外层循环只查 `exit_flag`，不检查 `pause_requested`。内层 EXP 遍历也不检查。暂停后消费者按正常速度处理队列条目（每条 x N 个 EXP，每次间隔 sleep_time）。
- 影响：暂停后要等几分钟到几十分钟才生效。
- 修复方向：暂停时跳过 `time.sleep(self.sleep_time)`，消费者继续排空队列（不丢资产），只是加速。**不需要提前 break，队列排空后才停。**（✅ 已定）

### BL-AUDIT-002 批量暂停状态被 `_finalize_run` 覆盖为"已完成"

- 文件：`batch_task_executor.py:255-264, 409-420`
- 现象：`run()` 的 finally 块先把 status 设为 4（暂停），紧接着 `_finalize_run()` 检测到 `is_over=True` + `dispatch_token is None`，把 status 覆盖成 1 + process="100%"。
- 修复方向：`_finalize_run` 开头加 `if self.pause_requested and not self.stop_requested: return`。（✅ 已定）

### BL-AUDIT-003 批量队列容量过大

- 文件：`batch_task_executor.py:147, 159-161`
- 修复方向：`_qsize = max(10, thread_num * 2)`，去掉 1000 下限。queue_output 也从 2 倍改为 1 倍。gevent 模式同样改。（✅ 已定）

### BL-AUDIT-004 批量 saveResult_thread 不是 daemon 线程

- 文件：`batch_task_executor.py:292`
- 修复方向：加 `daemon=True`，和同文件 producer、consumer 一致。（✅ 已定）

## gevent 子进程

### BL-AUDIT-011 清理 task operate 中 status="2"（停止）死代码

- 文件：`batch_exp_task.py:964-977`, `auto_scan_task.py:814-818`, `gevent_batch_runner.py`, 相关 `tasks.py` 和 `tests.py`
- 现象：前端不渲染"停止"按钮，`status="2"` 永不可达。连带 `ProcessTaskKiller` 类、`BATH_TASK_DIC`/`KILL_AUTO_TASK_DIC` 的 kill 调用均无实际触发路径。
- 修复方向：删除死代码，清理相关 import 和测试。（✅ 已定）

## 低优先级（建议记录，不立即修）

### BL-AUDIT-012 dirscan 僵尸恢复不覆盖 status=0（排队中）

- 文件：`apps.py:109-142`
- 现象：DirScanTask 只检查 status=1（运行中），不检查 status=0 + queued=True 的排队任务。worker 在 CAS 认领后、status=1 写入前崩溃 → 任务永远卡在 status=0。
- 影响：极端边缘场景，概率极低。

### BL-AUDIT-013 引擎 API 分页循环异常不终止

- 文件：`cyberspace_engine_service.py:135-146`
- 修复方向：
  1. 返回条数 < page_size 时正常结束（引擎无更多数据）
  2. 连续 3 次去重后新增加数 = 0 时异常退出（引擎在返回重复数据）
  不加总超时。（✅ 已定）
