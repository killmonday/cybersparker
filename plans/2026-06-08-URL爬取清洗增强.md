# URL爬取HTML清洗增强

## 问题

`crawl_urls.js` 用 turndown 默认配置转换 `page.content()` 全页 HTML → markdown，产出含大量 HTML 标签残留和页面噪音（导航/页脚/侧边栏）。

## 方案

在 `crawl_urls.js` 中新增三层处理：

1. **内容提取**：用 `page.evaluate()` 在浏览器上下文中执行 DOM 操作
   - 先移除通用噪音元素
   - 再按站点域名匹配专用选择器（GitHub/知乎/微信公众号）
   - 最后回退到通用内容查找
2. **turndown 自定义规则**：添加 table/kbd/details 等规则
3. **后清洗**：正则清除残留 HTML 标签、实体解码、空白合并

## 风险

- 站点 HTML 结构调整会导致选择器失效 → 回退到通用提取
- turndown 自定义 table 规则对复杂表格（colspan/rowspan）支持有限 → 标注为已知限制
- 不引入新 npm 依赖，table 转换用自实现

## 验证

- Node.js 语法检查：✅ `node --check scripts/crawl_urls.js` 通过
- Django 系统检查：✅ 0 issues
- TypeScript 类型检查：✅ tsc --noEmit 通过
- 逻辑审查：✅ escapeHtml 已内联到浏览器上下文，站点选择器覆盖 4 类目标平台
- 后续需在真实环境中用实际 URL 端到端测试

## 结果

三层清洗管线已部署到 `crawl_urls.js`：
1. 浏览器端内容提取（站点适配 + 噪音移除）
2. turndown 自定义规则（table / kbd / details / 空元素）
3. markdown 后清洗（去残留标签 + 实体解码 + 空白合并）

接口（stdin JSON → stdout JSON）不变，Python 调用端无需修改。
