# 测绘引擎非HTTP协议资产导入 — 开发计划

- 状态：已完成
- 日期：2026-06-18
- 关联 backlog：BL-AUTO-020

## 目标

自动扫描任务从测绘引擎（Fofa/ZoomEye/Quake）查询时，非 HTTP 协议资产（SSH/FTP/RDP 等）也能入库参与全局检索，不再被丢弃。

## 数据流

```
测绘引擎返回 [{host, ip, port, protocol="ssh"}]
  → adapter._build_preferred_target → "ssh://1.2.3.4:22"
  → normalize_target 保留原样
  → 写入目标文件
  → producer 读取 → queue_input
  → request_scan 收到 "ssh://1.2.3.4:22"
    → parsed.scheme = "ssh" (非 http/https)
    → 跳过 HTTP 请求
    → DNS 解析拿 IP
    → 返回 minimal extra (无 HTML/header/title/favicon/cert)
  → queue_fingerpoint_input
  → fingerpoint_consumer_thread → fingerprint matched=[]
  → save_indentify_to_db → auto_scan_indentify_result (protocol=ssh)
```

## 代码改动

### 1. `auto_exp_task.py` — `request_scan`

在 URL 解析后、HTTP 请求前，加非 HTTP 分支：

```python
# 非 HTTP 协议：不发 HTTP 请求，直接返回最小数据
if parsed.scheme and parsed.scheme not in ("http", "https"):
    extra = _build_non_http_extra(url, parsed)
    return None, None, None, None, extra, None
```

### 2. `auto_exp_task.py` — `save_indentify_to_db`

扩展端口映射：
```python
_NON_HTTP_DEFAULT_PORTS = {
    "ssh": 22, "ftp": 21, "rdp": 3389, "smb": 445,
    "telnet": 23, "mysql": 3306, "mssql": 1433,
    "redis": 6379, "mongodb": 27017, "postgres": 5432,
    "oracle": 1521, "postgresql": 5432,
}
```

## 风险

| 风险 | 等级 | 说明 |
|------|------|------|
| DNS 解析阻塞 asyncio event loop | 低 | 用 `asyncio.to_thread` 包装，已有类似代码可参考 |
| 测绘引擎返回的 protocol 字段不准确 | 低 | 依赖引擎数据质量，不做服务验证 |

## 不做

- 非 HTTP 协议的指纹匹配和漏洞扫描
- 协议级服务探测（不验证端口是否真的运行对应服务）
