# 文件托管 — 设计文档

> 状态：已完成 | 2026-06-19

## 做什么

在后台新增一个"文件托管"页面，用户可以上传任意文件，设置公开或需登录下载，生成直接下载链接。

## 数据流

### 上传流程

```
用户选文件 + 选"公开/需登录" → 表单提交
  → POST /api/v1/hosted-files/upload (multipart)
  → 校验：文件大小 ≤ 200MB
  → 生成磁盘文件名（UUID + 原扩展名）
  → 写入 ./upload_files/<stored_name>
  → 写入 HostedFile 表（original_name, stored_name, file_size, is_public）
  → 返回文件记录 → 前端刷新列表
```

### 下载流程

```
请求 GET /files/<stored_name>
  → AuthMiddleware 放行（/files/ 前缀）
  → 查 HostedFile 表 where stored_name = <stored_name>
  → 不存在 → 404
  → 存在：
    - is_public=True → 直接返回文件（Content-Disposition: attachment）
    - is_public=False → 检查 request.session["info"]
      - 已登录 → 返回文件
      - 未登录 → 403
```

### 重命名流程

```
用户在列表点"重命名" → 弹窗输入新名
  → PUT /api/v1/hosted-files/<id>/rename {new_name: "xxx.pdf"}
  → 查 HostedFile 记录
  → 磁盘文件 rename(旧stored_name, 新stored_name)
  → 更新 DB stored_name 字段
  → 返回更新后的记录 → 前端刷新列表
```

### 删除流程

```
用户在列表点"删除" → 确认弹窗
  → DELETE /api/v1/hosted-files/<id>
  → 查 HostedFile 记录
  → 删除磁盘文件 ./upload_files/<stored_name>
  → 删除 DB 记录
  → 前端刷新列表
```

### 修改访问级别流程

```
用户在列表切换"公开/需登录"开关
  → PUT /api/v1/hosted-files/<id>/access {is_public: true/false}
  → 查 HostedFile 记录
  → 更新 is_public
  → 返回更新后的记录
```

## 数据模型

```python
class HostedFile(models.Model):
    original_name = models.CharField(max_length=512)  # 用户上传的原始文件名
    stored_name = models.CharField(max_length=512, unique=True)  # 磁盘上的文件名
    file_size = models.BigIntegerField()  # 字节
    is_public = models.BooleanField(default=True)  # True=公开, False=需登录
    created_at = models.DateTimeField(default=timezone.now)
    updated_at = models.DateTimeField(auto_now=True)
```

## API 设计

### 管理 API（需登录，走 `/api/v1/` 由 AuthMiddleware 保护）

```
GET    /api/v1/hosted-files              → 文件列表
POST   /api/v1/hosted-files/upload       → 上传（multipart: file + is_public）
DELETE /api/v1/hosted-files/<id>         → 删除
PUT    /api/v1/hosted-files/<id>/rename  → 重命名 {new_name: str}
PUT    /api/v1/hosted-files/<id>/access  → 修改访问 {is_public: bool}
```

### 下载入口（不经过 `/api/v1/`，AuthMiddleware 放行，视图自己判断权限）

```
GET    /files/<id>/<原始文件名>            → 下载文件
```

## 前端页面

- 路由：`/react-shell/hosted-files`
- 组件：Table（文件列表）+ Upload（上传按钮 + 公开/鉴权选择）+ 操作列（下载/重命名/删除/切换访问级别）
- 侧边栏：一级菜单"文件托管"，图标 `CloudServerOutlined`

## 安全

- `/files/` 前缀在 AuthMiddleware 放行，但下载视图内自行判断 `is_public` / session
- 上传文件存储路径使用 `os.path.abspath` + 目录前缀校验，防止路径穿越
- 重命名时校验新文件名不含路径分隔符
- 下载时只从 `stored_name` 查 DB，不信任用户传入的任意路径参数

## 安全审计与修复

| 问题 | 等级 | 修复 |
|------|------|------|
| Content-Disposition 响应头注入（`\r\n"`） | CRITICAL | 拼 `filename="..."` 前剔除 `\r\n"` 字符 |
| `f.read()` 全量读入内存 | HIGH | 改用 `FileResponse` 流式传输 |
| 下载 ID 可枚举遍历 | HIGH | 额外校验 URL 文件名 == `record.original_name` |
| null 字节绕过重名校验 | MEDIUM | `'\x00' in new_name` 判黑 |
| 上传文件名无长度限制 | MEDIUM | `original_name` 截断到 256 字符 |
| 重命名不检查目标是否存在 | 低（撤回） | 磁盘名含 UUID 后缀，碰撞概率可忽略 |
