# 种子数据导入判断修复

## 问题

Docker 全新部署后指纹表和 PoC 表数据为空。

## 根因

`docker-entrypoint.sh` 第 22 行用 `psql -t` 查 `COUNT(*)`，psql 输出的是对齐格式（如 `     0`，多个前导空格）。第 23 行用字符串精确匹配 `"0"` 和 `" 0"`，都匹配不上 → 误判为"已有数据" → 跳过导入。

## 修复

1. `psql -t` → `psql -tA`：`-A` 是 unaligned 模式，输出纯 `0` 无前导空格
2. 字符串比较 `= "0"` → 整数比较 `-eq 0`：双重保险
3. 导入命令 `2>/dev/null` → `2>&1`：导入错误输出到容器日志，方便排查
4. 补充 SEED_FILE 不存在时的日志提示

## 验证

修完后新机器 `docker compose up -d` 日志应出现：
```
[entrypoint] importing seed data (fingerprint table is empty)...
[entrypoint] seed data imported successfully
```

## 已部署机器的补救

```bash
docker compose exec -T web psql -h postgres -U postgres cybersparker < deploy/seed/seed_data.sql
```

## 风险

无。改动仅影响 entrypoint 判断逻辑，不影响已导入数据的机器。
