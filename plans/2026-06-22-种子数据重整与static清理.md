# 参考数据 ID 重整 + static 历史残留清理

## 背景

1. 开发库多次清空参考表再重导种子数据，序列未重置导致 ID 膨胀（fingerprint 3.4 万行但 ID 最大 13 万）
2. `app_cybersparker/static/` 和 `./static/` 下有大量旧前端、历史 CSV、残留 favicon 文件
3. 需要在生产环境执行 ID 重整，并清理历史残留

## 做了什么

### ID 重整
- 创建 `deploy/seed/reset_ids.sql`：用 `session_replication_role='replica'` 禁用 FK 触发器 → ROW_NUMBER 生成映射 → ID 上移再归位规避碰撞 → 重置序列
- 重新导出 `seed_data.sql`（干净 ID，9.5M → 8.9M）
- 创建 `deploy/seed/export_seed_data.sh`（可复用导出脚本）

### static 清理
- 删除 `app_cybersparker/static/exports/`（146 个废弃 CSV）
- 删除 `static/project2/`（628 个旧 Bootstrap/jQuery 文件）
- 删除 `app_cybersparker/static/project2/`（废弃 JS + 误放的 .omc）
- flag.css 移到 `app_cybersparker/static/css/`，前端引用同步更新
- 删除 2 个测试旧 jQuery 代码的废弃测试

### 文档同步
- `docs/modules/06-运行部署与可观测性模块.md`：6.3 节补充全部部署脚本清单和静态文件约定
- `deploy/seed/README.md`：补充 export_seed_data.sh 和 reset_ids.sql 说明
- `CHANGELOG.md`：记录所有变更

## 生产环境使用

```bash
# ID 重整
docker compose exec -T postgres psql -U postgres cybersparker < deploy/seed/reset_ids.sql

# 更新种子数据
bash deploy/seed/export_seed_data.sh
```
