# BL-PLUGIN-010 插件调试页插件列表懒加载

## 问题

`GET /api/v1/exp-debug/plugins` 一次性 `EXP.objects.all()` 加载全部插件，前端页面打开时全量拉取。插件数量多（数百到数千条）时接口响应慢、前端渲染慢、网络传输大。

## 方案

后端加 `?q=`/`offset`/`limit` 参数支持分页，前端改为下拉打开时自动加载首批 + 滚到底加载更多 + 搜索时重置分页。

### 后端改动

`app_cybersparker/views/expload/exp_debug.py:debug_getPluginList`:
- `?q=`：按 title/CVE icontains 过滤，不传则查全部
- `?offset=` + `?limit=`：分页参数（limit 上限 100）
- 多取 1 条判断 `has_more`（比 COUNT 更轻量）
- 响应新增 `has_more: bool`

### 前端改动

`frontend/src/pages/ExpDebugPage.tsx`:
- `onDropdownVisibleChange`：下拉打开且列表为空时自动 `GET /api/v1/exp-debug/plugins?offset=0&limit=50`
- `onSearch`：输入关键词 300ms 防抖 → 重置 offset=0 → 搜索请求
- `onPopupScroll`：滚到底（`scrollTop+clientHeight >= scrollHeight-10`）→ offset+=50 → 追加请求
- `filterOption={false}`：搜索由服务端控制

## 验证

- Django check 0 issues ✅
- Vite build 通过 ✅

## 风险

- 低。不改 API 响应格式（只加了 has_more 字段），不改数据库，不改其他页面。
