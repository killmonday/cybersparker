# zones API 性能优化：?counts=1 时 COUNT(DISTINCT) 聚合优化

- 日期：2026-06-24
- 模块：扫描区域 (zone_manage)
- 状态：已完成

## 问题

`GET /api/v1/zones?counts=1` 在 5000 资产时响应 16 秒。原因：Django ORM 的 5 个 `Count(distinct=True)` 注解生成一条 SQL，对所有关联表做 `LEFT JOIN`，中间结果集组合爆炸。

举例：公网 zone (id=1) 有 5000 资产 × 100 自动扫描任务 × 50 批量任务 × 30 目录扫描任务 × 1000 目录结果 → JOIN 后中间行数 7.5×10¹¹。

## 方案

把 5 个 `COUNT(DISTINCT ...)` 注解拆成 5 条独立 `SELECT zone_id, COUNT(*) FROM table GROUP BY zone_id` 查询，Python 中合并结果。每条查询走 zone_id 索引，无 JOIN。

## 改动范围

- `app_cybersparker/views/expload/zone_manage.py`：重写 `zone_list_api` 的 `?counts=1` 分支

## 风险

低。API 返回格式不变，前端无需改动。
