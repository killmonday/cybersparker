# 复制 URL 剪贴板修复

## 问题

资产检索页面的"复制URL"按钮不起作用。`navigator.clipboard.writeText()` 只在安全上下文（HTTPS 或 localhost）下可用，本项目部署在 HTTP（Nginx 端口 28600），所以 `navigator.clipboard` 为 `undefined`，调用静默失败。

## 影响范围

前端 4 处使用了 `navigator.clipboard.writeText()`：
1. `GlobalAssetSearchPage.tsx:838` — 复制资产 URL
2. `PluginDetailPage.tsx:44` — 复制插件代码
3. `HostedFileListPage.tsx:113` — 复制文件下载链接
4. `PocGenTaskExecutePage.tsx:146` — 复制提示词

全部受影响。

## 修复方案

创建共享工具函数 `copyToClipboard` → 优先用 Clipboard API，失败时降级到 `document.execCommand('copy')`（兼容所有 HTTP 环境）。

## 修改文件

- 新建 `frontend/src/utils.ts` — 共享剪贴板工具函数
- 修改 4 个页面文件，替换 `navigator.clipboard.writeText()` 调用

## 风险

- 无。`execCommand('copy')` 是成熟降级方案，所有浏览器支持。
